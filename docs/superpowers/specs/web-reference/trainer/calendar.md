# Календарь тренера — CalendarPage.tsx (+ SessionsCalendar.tsx)

**Маршрут:** `/calendar` · **Точки входа:** плитка «Календарь» и герой с главной
**Назначение:** единый календарь всех занятий тренера (по всем клиентам) с видами день/неделя/месяц; создание, правка, удаление занятия; привязка тренировки-плана, выбор зала, статусы.

## Макет (сверху вниз)

1. **ScreenHeader** «Календарь», back → `/`.
2. **SessionsCalendar** (стартовый вид `week`, `anchor` поднят в страницу):
   - **Шапка периода:** `‹` / подпись периода / `›` + кнопка «Сегодня». Подпись: день — «Суббота, 21 июня»; неделя — «N–M месяц» (две недели, лента); месяц — «ИЮНЬ 2026».
   - **MonthView:** сетка 7×N, ячейка-день = число + бейджи счётчиков подтверждений (`pending` серым `ink-mutedxl`, `confirmed` `accent-text`, `declined` `danger`). Сегодня — `ring-accent`. Тап по дню → вид «День».
   - **WeekView:** бесконечная вертикальная лента недель (скользящее окно), липкая шапка дней. Колонка-день = стопка карточек занятий (цвет по статусу) + пустая зона (тап → создать на 09:00). Тап по числу → день; `onRangeChange` догружает занятия под окно.
   - **DayView:** часовая сетка (`CAL_START_HOUR`…), автоскролл к **07:00**, линия «сейчас» (коралловая). Блоки занятий позиционируются по времени/длительности, пересечения раскладываются по колонкам. Тап по слоту → создать (дата+час), тап по блоку → правка.
   - **Нижний переключатель вида** (фикс. по центру снизу): День / Неделя / Месяц (активный — `accent`).
3. **FAB «+»** (фикс. снизу-справа, `tile-shadow-primary`): создать занятие с датой = текущий `anchor`.
4. **TrainerSessionSheet** (bottom-sheet) — форма создания/правки (см. ниже).

## Цвета блока занятия (приоритет сверху вниз)

1. `status==='cancelled'` → серый, перечёркнут, `opacity-50`.
2. `clientConfirmation==='confirmed'` → лайм `#caff3a` («Согласовано», и для planned, и для completed).
3. `status==='completed'` → голубой `#46d4f0` («Проведено»).
4. `clientConfirmation==='declined'` → красный `#ff5a5a` («Отклонено»).
5. иначе → оранжевый `#ffab2e` («Запланировано»/ждёт ответа).

## Данные

- `useSessions(from, to)` → `GET /api/sessions?from&to`. Диапазон = объединение базового окна (anchor ±) и скользящего недельного `weekRange`.
- **Фильтр:** показываются только **офлайн** занятия — `list = sessions.filter(s => s.isOnline === false)` (онлайн в тренерском календаре не видны).
- `useClients()` → `GET /api/clients` — карта `clientId → "Имя Фамилия"` для меток блоков (`renderLabel = имя ?? title ?? 'Занятие'`).

### Поля SessionResponse (что приходит)

`id, clientId, workoutId|null, date(YYYY-MM-DD), startTime(HH:MM), durationMin, location|null, title|null, status(planned|completed|cancelled), isOnline, note|null, clientConfirmation(pending|confirmed|declined)`.

## Форма занятия — TrainerSessionSheet

Открывается для: пустого слота (`createAt` = дата+время), FAB (`'new'`, дата = anchor), существующего занятия (правка). При `fixedClient` (календарь конкретного клиента) поле клиента не редактируется.

### Поля формы

| Поле                            | Контрол                                                                                                                         | Правила                                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| **Клиент**                      | при создании — `ClientSearchSelect` (поиск по имени/контакту/тегу, только `active`); при правке/`fixedClient` — статичный текст | **необязателен** (можно занятие без клиента)                                                      |
| **Дата**                        | текст-маска `ДД.ММ.ГГГГ` (`maskDmy`) ↔ ISO (`dmyToIso`)                                                                         | обязательна, год 1900..2100, валидная дата → иначе «Дата в формате ДД.ММ.ГГГГ»                    |
| **Время**                       | текст-маска `ЧЧ:ММ` 24ч (`maskTime`)                                                                                            | HH≤23 MM≤59 → иначе «Время в формате ЧЧ:ММ»                                                       |
| **Название**                    | input, `maxLength=200`                                                                                                          | опц.                                                                                              |
| **Тренировка**                  | кнопка → `WorkoutPickerSheet`; чип с именем + «Изменить»/«✕»                                                                    | опц., см. ниже                                                                                    |
| **Длительность**                | нативный `<input type="time" step=300>`, трактуется как Ч:ММ длительности                                                       | по умолч. 60 мин; минимум 5 мин                                                                   |
| **Место**                       | чипы залов (`useGyms`) + чип «Online» (взаимоисключающие)                                                                       | выбор зала ставит `location=g.name, isOnline=false`; «Online» ставит `isOnline=true, location=''` |
| **Статус** (только правка)      | чипы Запланировано/Проведено/Отменено (`STATUS_LABEL`)                                                                          | —                                                                                                 |
| **Запомнить** (только создание) | тумблер в шапке                                                                                                                 | см. prefs                                                                                         |

### Привязка planned-workout (WorkoutPickerSheet)

`plannedWorkout` — одно из: `{kind:'existing', id, name}` (уже привязанная, имя из списка тренировок клиента), `{kind:'template', template}`, `{kind:'history', workout}`.

- Пикер показывает **Шаблоны** (`useTemplates` → `GET /api/workout-templates`) — всегда; **Историю клиента** (`useClientWorkouts(clientId)` → `GET /api/clients/:id/workouts`, фильтр не-черновики/не-активные, сорт. по дате) — только если клиент выбран. Поиск по названию (`rankBySearch`).
- Выбор **не создаёт тренировку сразу**. При сохранении занятия `resolveWorkoutId()`:
  - `existing` → возвращает `id`;
  - `template`/`history` → если клиент выбран, создаёт **черновик** тренировки клиенту `POST /api/clients/:id/workouts` и берёт его `id`; если клиента нет → `null`;
  - нет `plannedWorkout` → `null` (отвязать).
  - `bodyFromTemplate`: разворачивает подходы шаблона в плоские записи (planned\*). `bodyFromHistory`: берёт только выполненные (`done`) подходы прошлой тренировки (actual ?? planned); если выполненных нет — `null`.

## Действия (мутации)

- **Сохранить (создание):** `useCreateSession(clientId).mutate(...)`
  - **API:** `POST /api/sessions`, тело `{ clientId|null, date, startTime, durationMin, title|null, location|null, isOnline, workoutId|null }` → `{ session }`. `onSuccess` → `closeForm()`.
- **Сохранить (правка):** `useUpdateSession(clientId).mutate({ id, patch })`
  - **API:** `PATCH /api/sessions/:id`, тело-патч `{ date, startTime, durationMin, title|null, location|null, isOnline, status, workoutId|null }` → `{ session }`. `onSuccess` → `closeForm()`.
- **Удалить** (только правка, после `window.confirm('Удалить занятие?')`): `useDeleteSession(clientId).mutate(id)`
  - **API:** `DELETE /api/sessions/:id` → `{ ok }`. `onSuccess` → `closeForm()`.
- **Создание тренировки из шаблона/истории:** `POST /api/clients/:id/workouts` (тело `CreateWorkoutRequest`) → `{ workout }`, до основной мутации.
- **Инвалидации** (любая из трёх мутаций сессии): `['sessions']` (все списки) + `['clients', clientId, 'sessions']`.

## Состояния

- **error загрузки:** вместо календаря — «Не удалось загрузить занятия. Попробуйте обновить страницу.» (`role="alert"`).
- **Ошибки формы** (после submit, `showErrors`): подсветка `border-danger` + текст под полем (дата/время). Клиент-ошибки нет (поле необязательно).
- **mutationError:** «Не удалось сохранить. Попробуйте снова.».
- **pending:** кнопка submit = «…», disabled (учитывает и создание тренировки).
- **Нет залов:** подсказка «Залы можно добавить в профиле тренера».
- **История пуста / тренировка без упражнений:** строки disabled / тексты «У клиента пока нет проведённых тренировок».

## Навигация

back → `/` · тап по дню/числу → вид «День» (внутри компонента) · переходов наружу из формы нет.

## Бизнес-правила и edge-cases

- **Онлайн скрыт:** созданное «Online» занятие в тренерском календаре не показывается (`isOnline===false`-фильтр), но создать его можно.
- **«Запомнить» (prefs, localStorage `calendar_last_session_prefs`):** тумблер только при создании. Переключение сразу пишет флаг `remember`. Поля (clientId/durationMin/location/isOnline) сохраняются при **успешном создании** (если `remember`), иначе prefs сбрасываются в `EMPTY_PREFS`. При следующем создании поля предзаполняются (запомненный клиент — только если ещё `active`). Стор общий с клиентским календарём.
- **mutationClientId:** для инвалидаций/создания тренировки = `fixedClient.id ?? session.clientId ?? выбранный clientId`.
- **Привязка существующей тренировки:** при открытии правки `workoutId` резолвится в имя из `clientWorkouts` (`useEffect`), чтобы показать чип.
- **Дата↔ISO:** хранение ISO, ввод ДД.ММ.ГГГГ; невалидная дата (несуществующее число/год вне диапазона) → `dmyToIso=null` → ошибка.
- **Длительность:** `<input type="time">` ставит `[color-scheme:dark]`; пустое/0 → минимум 5 мин; формат подписи через `formatDuration` («1 ч», «90 мин» → «1 ч 30 мин»).
- **Раскладка пересечений** (день): кластеризация по времени, колонки по `layoutColumns` — одновременные занятия раздвигаются по ширине.
- **Скользящее окно недель:** старт 20 недель (4 до anchor), докуп/выброс чанками по 6 у краёв (порог 800px), потолок 44 недели в DOM; позиция прокрутки удерживается.

## Сводка эндпоинтов

- `GET /api/sessions?from&to` — занятия за диапазон (фильтр офлайн на клиенте).
- `POST /api/sessions` — создать; `{clientId?,date,startTime,durationMin,title?,location?,isOnline,workoutId?}` → `{session}`.
- `PATCH /api/sessions/:id` — обновить (+`status`) → `{session}`.
- `DELETE /api/sessions/:id` → `{ok}`.
- `GET /api/clients` — имена клиентов / список для выбора.
- `GET /api/gyms` — залы для чипов «Место».
- `GET /api/workout-templates` — шаблоны для пикера тренировки.
- `GET /api/clients/:id/workouts` — история тренировок клиента (пикер).
- `POST /api/clients/:id/workouts` — создать черновик тренировки из шаблона/истории при сохранении занятия.
