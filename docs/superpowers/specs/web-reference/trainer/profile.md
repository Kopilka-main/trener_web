# Профиль тренера — ProfilePage.tsx

**Маршрут:** `/profile` · **Точки входа:** главная тренера
**Назначение:** карточка тренера (имя, email, дата рождения, аватар), «о себе», залы (CRUD), контакты, уведомления, тема, ID/QR и выход; отдельный режим редактирования профиля с кроп-аватаром.

## Макет (сверху вниз) — режим просмотра

1. **`ScreenHeader`** «Профиль», назад → `/`; справа кнопка `Pencil` → режим правки.
2. **Карточка тренера** (`bg-card` rounded-3xl): `Avatar` 64×64 (фото `/api/files/<avatarFileId>?v=<avatarFileId>` или инициалы) + полное имя (19px bold) + email (mutedxl) + «Дата рождения: ДД.ММ.ГГГГ» (если задана). В просмотре аватар без загрузки.
3. **«О себе»** (`Section`, только если `bio`) — карточка-текст.
4. **`GymsSection`** «Залы» — см. ниже.
5. **«Контакты»**: первая строка — Email (иконка `Mail`, ссылка `mailto:`, `ChevronRight`); затем `contacts[]` (тип + значение).
6. **«Уведомления»** — компонент `NotificationsToggle` (push-тумблер).
7. **«Тема»** — `ThemeToggle` (две кнопки Светлая/Тёмная, активная `bg-accent`).
8. **`IdRow`** «ID пользователя» — см. ниже (копирование + QR).
9. **`LogoutButton`** «Выйти» (`text-danger`, иконка `LogOut`).

### Залы (`GymsSection`)

- Список залов (`bg-card`): имя + `HoldToDelete` (удерживать для удаления).
- Кнопка «Добавить зал» (пунктир) → инлайн-поле (autoFocus, placeholder «World Class, СССР…»); Enter или «Готово» создаёт, Escape отменяет; «Готово» `disabled` при пустом имени/`isPending`.

### ID / QR (`IdRow`)

- Кнопка-строка с `id` (mono) → копирование в буфер; иконка меняется `Copy`→`Check` на 1.5с, всплывает тост «Скопировано» (внизу экрана).
- Кнопка «Показать QR» / «Скрыть QR» → разворачивает `QRCodeSVG` (значение = `id`, 200px) на белом фоне + подпись «Отсканируйте, чтобы получить ID тренера».

## Макет — режим редактирования (`ProfileEdit`)

- `ScreenHeader` «Редактировать», назад = закрыть; справа «Сохранить» (`text-accent-text`, при `isPending` — «…»).
- **`AvatarEditor`** (по центру): `Avatar` 64×64 (тап → выбор файла) + ссылка «Изменить фото» (при загрузке «Загрузка…») + (если есть фото) «Удалить» (`Trash2`). После выбора файла — модалка **`AvatarCropper`**.
- **Имя** / **Фамилия** (`EditField`, 2 колонки).
- **Дата рождения** — маска `ДД.ММ.ГГГГ` (`inputMode=numeric`, `maxLength=10`), хранится ISO.
- **О себе** — textarea (4 строки), placeholder «Опыт, подход, специализация…».
- **Связь** — список контактов: ряд чипов-типов (Телефон/WhatsApp/Telegram/MAX/Instagram/Прочее), `X` удаляет контакт; при типе «Прочее» — доп. поле названия типа; поле значения. Кнопка «Добавить контакт» (новый по умолчанию «Телефон»).
- Ошибка валидации — `text-danger` (`role=alert`).

## Данные

| Поле      | Источник                                                      | Формат            |
| --------- | ------------------------------------------------------------- | ----------------- |
| `trainer` | `useMe()` → `GET /api/auth/me` → `{ trainer }`                | `TrainerResponse` |
| залы      | `useGyms()` → `GET /api/gyms` → `{ gyms }`                    | `GymResponse[]`   |
| тема      | `getTheme()` из `lib/theme` (localStorage)                    | `light`/`dark`    |
| push      | `NotificationsToggle` (`lib/push`, Service Worker / Web Push) | —                 |

**`TrainerResponse`**: `id`, `email`, `firstName`, `lastName`, `title` (`string|null`), `bio` (`string|null`), `birthDate` (`string|null`, ISO `YYYY-MM-DD`), `contacts: {type,value}[]`, `avatarFileId` (`string|null`).
**`GymResponse`**: `id`, `name`, `monthlyRent` (`number|null`), `note` (`string|null`), `createdAt`.
**`TrainerContact`**: `type` (1–40), `value` (1–200).
**Типы контактов (чипы):** Телефон, WhatsApp, Telegram, MAX, Instagram, Прочее.

**Тело `UpdateTrainerRequest`** (все поля опциональны): `firstName?` (1–100), `lastName?` (1–100), `title?` (`≤200`/`null`), `bio?` (`≤2000`/`null`), `birthDate?` (`^\d{4}-\d{2}-\d{2}$`/`null`), `contacts?` (`{type(1–40),value(1–200)}[]`, ≤20).
В UI при сохранении: имя/фамилия `.trim()` (обязательны, иначе «Имя и фамилия обязательны.»); пустой `bio` → `null`; пустая дата → `null`, иначе ISO через `displayToIso`; контакты тримятся, тип пустой → «Прочее», контакты с пустым значением отбрасываются.
**Тело `POST /api/gyms`** (`CreateGymRequest`): `name` (1–200), опц. `monthlyRent` (>0/null), `note` (`≤2000`/null) — в UI задаётся только `name`.

## Состояния

- `me.isPending` → «Загрузка…» (только заголовок).
- `!trainer` → «Не удалось загрузить профиль.» (`role=alert`).
- Просмотр vs Редактирование — локальный `editing`.
- Нет `bio` → блок «О себе» скрыт; нет `birthDate` → строка даты скрыта.
- Загрузка/удаление аватара (`busy`) → кнопки `disabled`/`opacity`.
- Сохранение профиля: «Сохранить» → «…» при `update.isPending`; ошибка → «Не удалось сохранить. Попробуйте снова.».
- Залы: список скрыт, если пуст; «Готово» `disabled` при пустом имени/`create.isPending`.
- `NotificationsToggle`: не поддерживается браузером → пояснение (на iPhone — добавить на «Домой»); `denied` → тумблер `disabled` + подсказка про настройки браузера; служебные заметки про «no-key»/«denied».
- Копирование ID: иконка `Check` и тост «Скопировано» на 1.5с.

## Навигация

`Pencil` → правка (локально, без роута) · Email → `mailto:` · «Выйти» → разлогин (роутинг по `['me']`). Внешних переходов между экранами нет.

## Бизнес-правила и edge-cases

- **Аватар тренера**: URL — `/api/files/<avatarFileId>?v=<avatarFileId>` (через общий files-эндпоинт, не отдельный auth-роут как у клиента), кэш-бастинг через `?v=`. Загрузка — отдельный `fetch` с `credentials:'include'`, поле `photo`, без ручного `Content-Type` (браузер ставит boundary). `AvatarCropper` отдаёт **квадратный JPEG** (`avatar.jpg`): фото двигается/масштабируется (zoom 1–4) в рамке 288px, круглая подсказка.
- **Дата рождения**: ввод маскируется `ДД.ММ.ГГГГ` (`maskDate`), валидируется (`displayToIso`: ровно 8 цифр, год 1900–2100, реальная дата), хранится/шлётся ISO `YYYY-MM-DD`; некорректная → `null`.
- **Контакты** редактируются целиком (полная замена массива в PATCH). Тип «Прочее» допускает произвольное название (отдельное поле); типы вне списка показываются как «Прочее».
- **Залы** — отдельные сущности (`/api/gyms`), CRUD здесь только создание и удаление (имя). `monthlyRent`/`note` в UI профиля не задаются.
- **Тема** — чисто клиентская (localStorage + класс на `<html>`), без API.
- **QR/ID** — клиентские (`qrcode.react`, `navigator.clipboard`), без API; QR кодирует `id` тренера для подключения клиента.
- Цвет: один acid-fill — активные кнопки/тумблер `bg-accent`. «Выйти» — `text-danger` (реальное действие); «Удалить» зал/аватар — нейтральные ink-токены.

## Сводка эндпоинтов

- `GET /api/auth/me` — текущий тренер (профиль).
- `PATCH /api/auth/me` — обновить профиль (имя/фамилия/title/bio/дата/контакты).
- `POST /api/auth/me/avatar` — загрузить аватар (multipart, поле `photo`).
- `DELETE /api/auth/me/avatar` — удалить аватар.
- `POST /api/auth/logout` — выход.
- `GET /api/gyms` — список залов.
- `POST /api/gyms` — создать зал.
- `DELETE /api/gyms/:id` — удалить зал.
- `GET /api/files/<avatarFileId>?v=<id>` — изображение аватара тренера.

## Расхождения мобайла (на момент составления)

- [P1] Полный цикл аватара: выбор → **кроп (квадрат, zoom/pan 1–4, рамка 288px)** → multipart-загрузка (`/api/auth/me/avatar`, поле `photo`) → удаление, с кэш-бастингом `?v=`. URL фото тренера — `/api/files/<id>`, не отдельный auth-роут.
- [P1] Залы как отдельный CRUD-раздел (`/api/gyms`): список с hold-to-delete + инлайн-добавление по имени.
- [P1] Редактор контактов: чипы-типы (Телефон/WhatsApp/Telegram/MAX/Instagram/Прочее) с произвольным названием для «Прочее» + добавление/удаление.
- [P2] ID/QR-блок (копирование + `QRCodeSVG` по `id`), маска даты рождения, переключатель темы, push-тумблер.
