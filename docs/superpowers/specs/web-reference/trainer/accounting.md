# Финансы тренера — AccountingPage.tsx

**Маршрут:** `/accounting` (заголовок «Финансы», `back="/"`) · **Точки входа:** главная тренера
**Назначение:** учёт доходов/расходов за выбранный период — сводка прибыли, списки операций с фильтрами по категориям/тегам, создание и удаление операций; продажи пакетов попадают сюда как синтетические доходы (`pkg:`).

## Макет (сверху вниз)

1. **`ScreenHeader`** «Финансы», стрелка назад → `/`.
2. **Блок управления периодом** (`px-2 pt-1`, `flex-col gap-2`):
   - **`RangePresets`** — пилюли в одну строку (`bg-chip`, активная `bg-accent text-accent-on`): **Месяц / Квартал / Год / Период**.
   - Ниже либо:
     - **`PeriodSwitcher`** (для месяц/квартал/год): `ChevronLeft` · числовые ячейки `SpinCell` · `ChevronRight`.
       - месяц → ячейки **ММ** + **ГГГГ**; квартал → **квартал(1–4)** + **ГГГГ**; год → **ГГГГ**.
       - `SpinCell` — ручной ввод цифр + стрелки ↑/↓ (клавиатура) для ±1, Enter = blur, на blur клампится в [min,max].
     - либо **`CustomDateRange`** (для «Период»): два `type=date` поля «С» / «По».
3. **Вкладки** (`grid-cols-3`, `bg-chip`, активная `bg-card text-ink`): **Сводка / Доходы / Расходы**.
4. **Контент вкладки** (`px-2 pb-10 pt-3`).

### Вкладка «Сводка» (`SummaryTab`)

- Большая карточка «Прибыль за период»: число 40px tabular (`balance`); знак `−` и цвет `--color-coral` при отрицательном, иначе `--color-accent`.
- Две карточки рядом: **Доходы** `+<totalIncome>` (`--color-accent-text`) и **Расходы** `−<totalExpense>` (ink).

### Вкладки «Доходы» / «Расходы» (`IncomeTab` / `ExpensesTab` → общий `EntriesView`)

1. **Поиск** (`bg-chip`, иконка `Search`): «Поиск по тегу или типу»; крестик `X` очищает.
2. **Подсказки тегов** (только при пустом поиске, если теги есть): чипы `#тег` со счётчиком; тап — установить/снять фильтр `#тег`.
3. **Чипы-категории** (`FilterChip`): «Все» + по одной на каждую встретившуюся категорию, со счётчиком; тап — фильтр по категории (повторный тап снимает).
4. **Карточка итога**: «Всего за период» или «`<фильтр>` за период», сумма 28px (`+`/`−`, цвет: доходы accent, расходы ink).
5. **Кнопка добавления** (пунктир, иконка `Plus`): «Добавить доход» / «Добавить расход» → разворачивает `EntryForm`.
6. **Список операций**, сгруппированный по датам (заголовок-дата «3 июня», год добавляется если не текущий). Строка `EntryRow` — см. ниже.

## Данные

| Поле                                   | Источник                                                                                | Формат              |
| -------------------------------------- | --------------------------------------------------------------------------------------- | ------------------- |
| период `from`/`to`                     | локальный стейт + `computePeriod(month, range)`; для «custom» — `customFrom`/`customTo` | ISO `YYYY-MM-DD`    |
| `totalIncome`/`totalExpense`/`balance` | `useAccountingSummary(from,to)` → `GET /api/accounting/summary?from&to`                 | число `formatMoney` |
| доходы                                 | `useIncomes()` → `GET /api/incomes` (весь список, фильтр по дате на клиенте)            | `IncomeResponse[]`  |
| расходы                                | `useExpenses()` → `GET /api/expenses` (весь список, фильтр по дате на клиенте)          | `ExpenseResponse[]` |
| имя клиента в строке                   | `useClients()` → `GET /api/clients`, карта `clientId → "Имя Фамилия"`                   | —                   |

**`IncomeResponse`**: `id`, `category`, `amount`, `date` (ISO), `clientId` (`string|null`), `note` (`string|null`), `tags: string[]`, `title` (`string|null`), `subtitle` (`string|null`), `createdAt`. Поля `title`/`subtitle` непусты только у синтетических строк-пакетов (`title` = тип тренировки, `subtitle` = напр. «20 трен.»).
**`ExpenseResponse`**: то же, плюс `gymId` (`string|null`); без `title`/`subtitle`.
**`AccountingSummaryResponse`**: `from`, `to`, `totalIncome`, `totalExpense`, `balance`.

**Модель строки `Entry`** (для `EntryRow`): `id`, `date`, `clientName` (из карты по `clientId` или `null`), `typeLabel` (= `category`), `title`, `subtitle`, `note`, `tags`, `amount`, `positive`, `onDelete?`.

**Категории (фиксированные, для формы):**

- Доход: **Тренировка, Консультация, Фарма, Прочее**.
- Расход: **Аренда, Инвентарь, Обучение, Фарма, Прочее**.

**`EntryRow`** (детальная плитка): первичный заголовок = `clientName ?? title ?? typeLabel`; кикер-тип показывается, если первичный заголовок ≠ типу; если есть и клиент, и `title` — `title` отдельной строкой; мета-строка `subtitle · note`; чипы `#тег` (тап → фильтр по тегу); сумма mono (`+`/`−`, accent/ink); `HoldToDelete` (если есть `onDelete`).

## Действия

| Жест/кнопка                                           | Эффект                                                                   | API (метод путь, тело → ответ)                                                        | Инвалидация                                          |
| ----------------------------------------------------- | ------------------------------------------------------------------------ | ------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| Пресет периода                                        | сменить `range`                                                          | —                                                                                     | —                                                    |
| `SpinCell` ±/ввод, стрелки `‹ ›`                      | сдвиг периода (`shiftPeriod`: год=±12 мес, квартал=±3 мес, месяц=±1)     | —                                                                                     | —                                                    |
| Вкладки                                               | переключить Сводка/Доходы/Расходы                                        | —                                                                                     | —                                                    |
| Поиск по тегу/типу                                    | локальная фильтрация (`matchesTagSearch` — по тегу, категории или title) | —                                                                                     | —                                                    |
| Чип-категория / чип-тег                               | установить/снять фильтр (`matchesFilter`: `#…` = тег, иначе категория)   | —                                                                                     | —                                                    |
| «Добавить доход»                                      | развернуть `EntryForm kind=income`                                       | —                                                                                     | —                                                    |
| «Добавить расход»                                     | развернуть `EntryForm kind=expense`                                      | —                                                                                     | —                                                    |
| «Сохранить» (доход)                                   | создать доход                                                            | `POST /api/incomes`, тело `CreateIncomeRequest` → `{ income }`                        | `['accounting','incomes']`                           |
| «Сохранить» (расход)                                  | создать расход                                                           | `POST /api/expenses`, тело `CreateExpenseRequest` → `{ expense }`                     | `['accounting','expenses']`                          |
| `HoldToDelete` на обычной операции (доход)            | удалить доход                                                            | `DELETE /api/incomes/:id` → `{ ok:true }`                                             | `['accounting','incomes']`                           |
| `HoldToDelete` на обычной операции (расход)           | удалить расход                                                           | `DELETE /api/expenses/:id` → `{ ok:true }`                                            | `['accounting','expenses']`                          |
| `HoldToDelete` на строке-пакете (`pkg:` + `clientId`) | удалить проданный пакет                                                  | `DELETE /api/clients/:clientId/packages/:pid` (`pid` = `id.slice(4)`) → `{ ok:true }` | `['accounting']` + `['clients',clientId,'packages']` |
| Тап по чипу `#тег` в строке                           | установить фильтр по этому тегу                                          | —                                                                                     | —                                                    |

**Тело `CreateIncomeRequest`**: `category` (1–200), `amount` (>0), `date` (`^\d{4}-\d{2}-\d{2}$`), `clientId?` (`string|null`), `note?` (`≤2000` или `null`), `tags?` (`string[]`, каждый 1–40, ≤30). В UI форме `clientId` не задаётся (всегда без клиента).
**Тело `CreateExpenseRequest`**: то же + `gymId?` (`string|null`); в UI форме не задаётся.
Валидация формы перед отправкой: `amount` (запятая → точка) должна быть конечной и > 0 («Укажите сумму больше нуля.»); `date` — формат `YYYY-MM-DD` («Укажите дату.»). Пустая заметка → `null`.

## Состояния

- **Сводка**: `isPending` → «Загрузка…»; ошибка/нет данных → «Не удалось загрузить сводку.» (`role=alert`).
- **Списки**: пусто без фильтра → «Нет доходов за период» / «Нет расходов за период»; пусто при активном фильтре/поиске → «Нет операций по фильтру».
- Кнопка «Сохранить» в форме `disabled` при `mutation.isPending` («Сохранение…»).
- Строка-пакет (`id` начинается с `pkg:`) без `clientId` → удаление недоступно (`onDelete` отсутствует, `HoldToDelete` не рендерится).

## Навигация

Внутриэкранная: только переключение `range`/период/вкладок и разворачивание формы. Внешних переходов нет (стрелка назад `ScreenHeader` → `/`).

## Бизнес-правила и edge-cases

- **Период считается на клиенте** (`computePeriod`): год = `YYYY-01-01…YYYY-12-31`; квартал = первый месяц квартала … последний день третьего месяца; месяц = `YYYY-MM-01 … последний день`. Операции фильтруются по `i.date >= from && i.date <= to` (даты от API уже по убыванию → группы по датам идут сверху вниз).
- **Бэкенд не фильтрует доход/расход по диапазону на сервере для этих списков** — грузится весь список тренера, фильтр по дате и клиенту делается на клиенте. Ключи кэша общие: `['accounting','incomes']`, `['accounting','expenses']`.
- **Синтетические строки-пакеты** (`pkg:<pid>`): продажа пакета занятий отражается доходом с заполненными `title`/`subtitle`; удаляется не как доход, а как пакет клиента (`DELETE …/packages/:pid`), что инвалидирует и бухгалтерию, и пакеты клиента.
- **Теги**: вводятся через `TagInput` (Enter/запятая добавляют, ведущий `#` отбрасывается, дубли игнорируются, Backspace на пустом удаляет последний); в фильтре префикс `#`.
- **Счётчики чипов**: категории и теги собираются из набора операций периода (`countBy`), сортируются по алфавиту.
- Цвет: один acid-fill — активная пилюля/чип/кнопка `bg-accent`. Ошибки формы — `text-danger` (текст об ошибке валидации, не действие).

## Сводка эндпоинтов

- `GET /api/accounting/summary?from&to` — прибыль/доходы/расходы за период.
- `GET /api/incomes` — все доходы тренера.
- `POST /api/incomes` — создать доход.
- `DELETE /api/incomes/:id` — удалить доход.
- `GET /api/expenses` — все расходы тренера.
- `POST /api/expenses` — создать расход.
- `DELETE /api/expenses/:id` — удалить расход.
- `DELETE /api/clients/:clientId/packages/:pid` — удалить проданный пакет (строка `pkg:`).
- `GET /api/clients` — имена клиентов для подписи операций.
