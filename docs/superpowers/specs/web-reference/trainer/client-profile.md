# Профиль клиента (просмотр) — ClientProfilePage.tsx

**Маршрут:** `/clients/:id/profile` · **Точки входа:** плитка «Профиль» карточки клиента
**Назначение:** read-only просмотр данных клиента (только заполненные поля); правка — отдельной кнопкой.

## Макет (сверху вниз)

1. **Кнопка «Править»** (вверху справа, `bg-card`, accent-текст) → `/clients/:id/edit`.
2. **Шапка по центру:** Avatar 96×96 (`/api/files/:avatarFileId` или инициалы) + имя
   «Имя Фамилия» (24px bold) + строка чипов: чип формата «Онлайн»/«Спортзал» (`isOnline`)
   - возраст «35 лет» (если есть `birthDate`).
3. **Раздел «Данные»** (если есть контакты/дата/связь) — сгруппированная карточка строк `DataRow`:
   - контакты (только с непустым `value`); если контактов нет, но есть `phone` — одна строка «Телефон».
   - дата рождения «11.06.1990» (иконка CalendarDays).
   - «Клиентский ID» = `accountId` (иконка Link2), тап копирует.
4. **Раздел «Теги»** (если есть): чипы `bg-chip` с префиксом `#`.
5. **Раздел «Заметки»** (если есть): текст `whitespace-pre-wrap`.
6. **Тост «Скопировано»** (1.5с) при долгом нажатии на строку.

### Строка данных (`DataRow`)

Иконка по типу + подпись (тип) + значение. Если есть `href` — вся строка кликабельна
(значение accent-цветом, внешние ссылки — в новой вкладке). Долгое нажатие (~600мс) копирует значение.

**`contactHref` по типу:** Телефон→`tel:`; WhatsApp→`https://wa.me/<цифры>`; Email→`mailto:`;
Telegram→`https://t.me/<handle>`; Instagram→`https://instagram.com/<handle>`;
ВКонтакте→`https://vk.com/<handle>` (URL пропускается как есть); прочее → не кликабельно.

## Данные

| Поле              | Источник                                                   | Формат              |
| ----------------- | ---------------------------------------------------------- | ------------------- |
| клиент            | `useClient(id)` → `GET /api/clients/:id` → `{ client }`    | `ClientResponse`    |
| формат            | `c.isOnline`                                               | «Онлайн»/«Спортзал» |
| возраст           | `ageFromBirthDate(c.birthDate)` + склонение                | «35 лет»            |
| `displayContacts` | `c.contacts.filter(value≠'')`, иначе `phone` → одна строка | —                   |
| дата              | `birthDisplay(c.birthDate)`                                | «ДД.ММ.ГГГГ»        |
| accountId         | `(c.accountId ?? '').trim()`                               | строка              |

## Действия

- Тап «Править» → `navigate('/clients/:id/edit')`.
- Короткий тап по кликабельной строке → `tel:`/`mailto:`/внешняя ссылка.
- Долгое нажатие (~600мс) по строке → копирование значения в буфер + тост «Скопировано».
- Тап по «Клиентский ID» (`tapCopy`) → копирование.
- Никаких мутаций (только чтение + копирование + переход в правку).

## Состояния

- **loading** (`client.isPending`): «Загрузка…».
- **error** (`client.isError || !client.data`): «Не удалось загрузить клиента.» (role=alert).
- Разделы «Данные»/«Теги»/«Заметки» рендерятся только при наличии данных.
- Возраст показывается только при валидном `birthDate`.

## Навигация

«Править»→`/clients/:id/edit` · контакты→внешние `tel:`/`mailto:`/`https:` · назад (хедер раздела)→карточка.

## Бизнес-правила и edge-cases

- Показываются **только заполненные** поля (read-only витрина).
- Долгое нажатие гасит переход по `href` (флаг `longPressed`), чтобы копирование не открывало ссылку.
- Внешние ссылки — `target=_blank rel=noopener noreferrer`.
- «Онлайн» — формат работы клиента (`isOnline`), не presence.

## Сводка эндпоинтов

- `GET /api/clients/:id` → `{ client }` — данные клиента.
- `GET /api/files/:avatarFileId` — аватар.
