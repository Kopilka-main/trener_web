# Чат с клиентом (тренер) — ClientChatPage.tsx

**Маршрут:** `/clients/:id/chat` · **Точки входа:** список диалогов `/messages`,
карточка клиента `/clients/:id`
**Назначение:** переписка тренера с клиентом — отправка (текст / задачи `/task` /
закреп `/pin`), ответ-цитата свайпом, закреплённые сообщения, статус прочтения,
удаление переписки.

## Макет (сверху вниз)

1. **ScreenHeader** — имя клиента (или «Чат»), назад → `/clients/:id`, справа —
   **HoldToDelete** (корзина, удержание): удалить переписку (история сотрётся у
   обоих, необратимо).
2. **Баннер закреплённых** (если есть `pinnedMessages`): иконка Pin + «Закреплённое
   · i/N» + текст текущего. Тап — переход к сообщению + переключение на следующее
   (цикл, как в Telegram). X — открепить показанное.
3. **Лента сообщений** (scroll, автоскролл вниз при новых):
   - **Свайп влево** по сообщению → ответить (`SwipeToReply`, порог 40px;
     вертикальный скролл не перехватывается; для `kind==='system'` отключён).
   - **Пузырь** (`Bubble`):
     - `kind==='system'` — серая плашка по центру (например «задача выполнена»).
     - `kind==='task'` — карточка справа с чекбоксом (read-only для тренера —
       закрывает клиент), кикер «Задача» / «Задача · выполнена», текст
       (зачёркнут+muted если done), время.
     - обычный — пузырь: свои (`senderRole==='trainer'`) акцентные справа, чужие
       `bg-card` слева. Над текстом — `ReplyQuote` (если есть `replyTo`, тап →
       переход к оригиналу). Время + статус **только на своих**: ✓ отправлено /
       ✓✓ прочитано клиентом.
   - Подсветка пузыря (`bg-accent/10`) на ~1.6с при переходе к нему.
4. **Панель ответа** (если `replyTo`): «Ответ на ваше / клиенту» + цитата, X — отмена.
5. **Поле ввода** (textarea, авто-рост до 3 строк/80px, maxLength 4000) + круглая
   акцентная кнопка отправки (ArrowUp), видна только при непустом тексте.

## Данные

| Поле               | Источник                                                                        | Формат                |
| ------------------ | ------------------------------------------------------------------------------- | --------------------- |
| сообщения          | `useChatMessages(id)` → `GET /clients/:id/messages` (polling 4с) → `messages[]` | `MessageResponse[]`   |
| `clientLastReadAt` | из того же ответа — момент прочтения **клиентом**                               | для статуса ✓✓        |
| `pinnedMessages`   | из того же ответа (опционально) — закреплённые, по возрастанию                  | массив                |
| имя в заголовке    | `useClient(id)` → `GET /clients/:id`                                            | «Имя Фамилия» / «Чат» |
| `m.kind`           | `text` / `task` / `system`                                                      | вид пузыря            |
| `m.taskDone`       | статус задачи (true=закрыта) или null                                           | чекбокс               |
| `m.replyTo`        | цитата `{id,senderRole,body}` или null                                          | ReplyQuote            |
| статус прочтения   | `read = clientReadAt!==null && m.createdAt <= clientReadAt`                     | ✓ / ✓✓                |

## Действия

| Жест/кнопка                               | Эффект                                            | API                                                                                                           |
| ----------------------------------------- | ------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| Отправить (Enter-кнопка / submit)         | Послать сообщение, очистить поле, сохранить фокус | `POST /clients/:id/messages` тело `{body, replyTo?}` → `{message}` · инвалидирует `['clients',id,'messages']` |
| Свайп влево по сообщению                  | Установить `replyTo`, фокус в поле                | — (затем уходит в `POST` как `replyTo`)                                                                       |
| X в панели ответа                         | Сбросить `replyTo`                                | —                                                                                                             |
| Тап по `ReplyQuote`/баннеру               | Скролл к оригиналу + подсветка                    | —                                                                                                             |
| X на баннере закреплённых                 | Открепить показанное                              | `DELETE /clients/:id/messages/:mid/pin` → `{ok}` · инвалидирует ленту                                         |
| Удержание корзины (header)                | Удалить переписку → `/messages`                   | `DELETE /conversations/:id` → `{ok}` · инвалидирует ленту, `['conversations']`, `['chat','unread']`           |
| Открытие чата / новое входящее от клиента | Отметить диалог прочитанным                       | `POST /clients/:id/messages/read` → `{ok:true}` · инвалидирует `['conversations']`                            |

**Команды в тексте (server-side):** `/task <текст>` — создаёт задачу с чекбоксом
для клиента; `/pin` — закрепляет сообщение (видно обоим). Подсказки показаны в
пустом состоянии. Парсинг — на сервере (фронт шлёт сырой `body`).

При ошибке отправки текст возвращается в поле (тренер не теряет сообщение).

## Состояния

- **loading** (`messages.isPending`): «Загрузка…».
- **error** (`messages.isError`): «Не удалось загрузить сообщения…» (role=alert).
- **empty** (`isSuccess && messages.length===0`): «Сообщений пока нет. Напишите
  первым.» + подсказки про `/task` и `/pin`.
- **есть закреплённые:** баннер сверху (с пагинацией i/N при нескольких).
- **режим ответа:** панель ответа над полем ввода, у формы убирается верхняя рамка.
- Кнопка отправки скрыта при пустом `draft`, disabled при `send.isPending`.

## Навигация

назад → `/clients/:id` · после удаления переписки → `/messages`.

## Бизнес-правила и edge-cases

- **Прочитано клиентом (✓✓):** `m.createdAt <= clientLastReadAt`; иначе ✓. Статус
  только на тренерских сообщениях.
- **Mark-read триггерится** при открытии и при каждом НОВОМ входящем от клиента
  (отслеживается `lastClientMsgId`), пока чат открыт — иначе у клиента не появится
  ✓✓, а у тренера «зависнут» непрочитанные.
- **Закреплённые** общие для обоих; баннер циклически переключает показываемое
  тапом; X открепляет текущее (не все).
- **Задача** для тренера — read-only индикатор; закрывает её клиент со своей стороны.
- Polling ленты 4с (вебсокетов нет); автоскролл вниз по росту длины списка; при
  фокусе поля лента ~500мс прижимается к низу каждый кадр (клавиатура не дёргает).
- textarea: maxLength 4000, авто-рост до 80px, далее внутренний скролл.
- Один acid-fill: акцент на своих пузырях + кнопке отправки.

## Сводка эндпоинтов

- `GET /clients/:id/messages` — лента + `clientLastReadAt` + `pinnedMessages` (polling 4с).
- `POST /clients/:id/messages` — отправить (`{body, replyTo?}`); `/task`, `/pin` парсятся сервером.
- `POST /clients/:id/messages/read` — отметить прочитанным тренером.
- `DELETE /clients/:id/messages/:mid/pin` — открепить сообщение.
- `DELETE /conversations/:id` — удалить всю переписку.
- `GET /clients/:id` — имя клиента для заголовка.
