# Календарь клиента (тренер) — ClientCalendarPage.tsx

**Маршрут:** `/clients/:id/calendar` · **Точки входа:** карточка клиента `/clients/:id`
**Назначение:** календарь занятий ОДНОГО клиента глазами тренера — просмотр (день/неделя/месяц), создание и редактирование занятий; клиент в форме зафиксирован.

## Макет (сверху вниз)

1. **ScreenHeader** — заголовок «Календарь · <Имя Фамилия>» (или «Календарь», пока
   клиент грузится), кнопка назад → `/clients/:id`.
2. **SessionsCalendar** (общий компонент) с `defaultView="month"`, `showStats`:
   - переключатель вида **День / Неделя / Месяц** (внизу, `ViewSwitcher`);
   - сетка слотов по часам (день/неделя) и сетка дат (месяц);
   - метки занятий = `s.title` или имя клиента (для инициалов в недельном виде);
   - **StatsBar** под календарём — счётчики по статусам (planned / confirmed /
     completed / declined / cancelled) для видимого набора.
   - Автоскролл к 7:00 в день/неделя виде.
3. **FAB «+»** (fixed, низ-справа, `tile-shadow-primary`, круг 56px) — создать
   занятие без предзаполненного слота.
4. **TrainerSessionSheet** (шторка-форма, общая с тренерским календарём) —
   открывается поверх при создании/редактировании. `fixedClient={{ id, name }}` —
   выбор клиента недоступен, список клиентов пуст.

## Данные

| Поле                  | Источник                                                                          | Формат                   |
| --------------------- | --------------------------------------------------------------------------------- | ------------------------ |
| занятия клиента       | `useClientSessions(id)` → `GET /sessions` + фильтр `s.clientId === id` на клиенте | массив `SessionResponse` |
| имя в заголовке/метке | `useClient(id)` → `GET /clients/:id` → firstName/lastName                         | строка                   |
| статистика StatsBar   | те же занятия, бакеты по `status`/`clientConfirmation`                            | счётчики                 |

> Важно: API `/sessions` фильтрует только по датам, поэтому отбор по клиенту
> делается **на фронте** (`select`). Диапазон не передаётся — тянется весь список.

## Действия

| Жест/кнопка                        | Эффект                                                                   | API                                                                                                                                             |
| ---------------------------------- | ------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Тап по пустому слоту (день/неделя) | Открыть форму создания с предзаполненными `date` + `startTime` (`HH:00`) | — (форма)                                                                                                                                       |
| Тап по занятию                     | Открыть форму редактирования этого занятия                               | — (форма)                                                                                                                                       |
| FAB «+»                            | Открыть форму создания без слота (дата = текущий anchor)                 | — (форма)                                                                                                                                       |
| Сохранение в форме (создание)      | Создать занятие                                                          | `POST /sessions` (тело `CreateSessionRequest`, `clientId` фиксирован) → `{session}` · инвалидирует `['sessions']` и `['clients',id,'sessions']` |
| Сохранение в форме (правка)        | Обновить занятие                                                         | `PATCH /sessions/:sid` (тело `UpdateSessionRequest`) → `{session}` · те же инвалидации                                                          |
| Удаление в форме                   | Удалить занятие                                                          | `DELETE /sessions/:sid` → `{ok}` · те же инвалидации                                                                                            |
| Смена вида День/Неделя/Месяц       | Локальный стейт компонента                                               | —                                                                                                                                               |

> Мутации create/update/delete живут в `TrainerSessionSheet` (см. тренерский
> `calendar.md`), здесь экран только передаёт `fixedClient`.

## Состояния

- **loading:** `sessions` грузится — календарь рисуется на пустом списке.
- **error:** `sessions.isError` → текст «Не удалось загрузить занятия. Попробуйте
  обновить страницу.» (ink-muted, role=alert) вместо календаря.
- **форма закрыта/открыта:** `editing` (null / 'new' / `SessionResponse`) и
  `createAt` управляют видимостью `TrainerSessionSheet`.
- Заголовок до загрузки клиента — просто «Календарь».

## Навигация

назад → `/clients/:id`. Форма — оверлей, без смены маршрута.

## Бизнес-правила и edge-cases

- Клиент в форме **зафиксирован** (`fixedClient`) — нельзя переназначить занятие
  другому клиенту с этого экрана.
- Слот по тапу → `startTime = "<HH>:00"` (часы из ячейки, минуты обнуляются).
- `defaultDate` формы = дата слота, иначе текущий `anchor` (видимый период).
- Онлайн-занятия в тренерском календаре по статистике учитываются как обычные
  (фильтрация «онлайн не в календаре тренера» — отдельное правило главного
  тренерского календаря, здесь календарь персональный по клиенту).

## Сводка эндпоинтов

- `GET /sessions` — все занятия тренера (фильтр по клиенту на фронте).
- `GET /clients/:id` — клиент (имя для заголовка/метки).
- `POST /sessions` — создать занятие.
- `PATCH /sessions/:sid` — обновить занятие.
- `DELETE /sessions/:sid` — удалить занятие.
