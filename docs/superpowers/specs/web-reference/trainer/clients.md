# Клиенты тренера — ClientsPage.tsx

**Маршрут:** `/clients` · **Точки входа:** плитка/раздел «Клиенты» с главной тренера, `back="/"`
**Назначение:** список клиентов тренера с поиском, сортировкой, фильтром по формату и FAB добавления.

## Макет (сверху вниз)

1. **ScreenHeader** `title="Клиенты"`, кнопка назад → `/`.
2. **Поиск** (`shelf`, rounded-2xl): иконка Search слева, `placeholder="Поиск по имени, тегу"`,
   крестик X справа для очистки (виден только при непустом запросе).
3. **Панель сортировки + фильтр** (под поиском, `flex-wrap gap-2`):
   - **Кнопка-тумблер сортировки** (pill `bg-card`, 1px line): иконка + подпись.
     - В режиме `alpha` показывает «По занятию» (иконка CalendarClock) — тап переключит на сорт. по занятию.
     - В режиме `session` показывает «По алфавиту» (иконка ArrowDownAZ) — тап вернёт алфавит.
   - **Сегмент-фильтр формата** (pill-группа `bg-card`): «Все» / «Онлайн» / «Спортзал».
     Активный сегмент — `bg-accent text-accent-on` (единственный acid-fill экрана).
4. **Список** (зависит от режима сортировки):
   - **`alpha`**: плоский `<ul>` строк `ClientRow`, отсортированных по «Имя Фамилия» (locale `ru`).
   - **`session`**: группы по дате ближайшего занятия (заголовок mono-uppercase
     «Сегодня» / «Завтра» / «3 июня»), затем группа **«Без занятий»**. В этом режиме
     строка показывает **время** занятия вместо даты.
5. **FAB** (sticky, низ-справа, `tile-shadow-primary`, круг 56×56, иконка Plus) → `/clients/new`.

### Строка клиента (`ClientRow`)

`Link` → `/clients/:id`, `bg-card` rounded-2xl, `row-glow`:

- **Avatar** 44×44 (фото `/api/files/:avatarFileId` или инициалы). У архивных — `muted`.
- Имя «Имя Фамилия» (15px semibold).
- Подстрока (mono 12px): время/дата ближайшего занятия → иначе телефон → иначе «без телефона».
- ChevronRight справа.
- Архивный клиент (`status==='archived'`): вся строка `opacity-60`.

## Данные

| Поле              | Источник                                                                                                                                | Формат             |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------- | ------------------ |
| список клиентов   | `useClients()` → `GET /api/clients` → `clients[]`                                                                                       | `ClientResponse[]` |
| фильтр поиска     | по `firstName lastName phone tags` (lowercase, `includes`)                                                                              | —                  |
| фильтр формата    | `typeFilter` против `c.isOnline` (`online`→true, `gym`→false)                                                                           | —                  |
| ближайшее занятие | `useSessions(today)` → `GET /api/sessions?from=<today>` → сессии `status==='planned'`, `date>=today`, минимум по (date,time) на клиента | дата/время         |
| архив-метка       | `c.status==='archived'` → `opacity-60`                                                                                                  | —                  |

`ClientResponse`: `id, firstName, lastName, phone|null, notes|null, accountId|null,
birthDate|null, status('active'|'archived'), contacts[], tags[], isOnline, avatarFileId|null, createdAt`.

## Действия

- Ввод в поиск → локальная фильтрация (без запроса).
- Тап по тумблеру → переключение `alpha`↔`session`.
- Тап по сегменту → `typeFilter` (`all`/`online`/`gym`), локальная фильтрация.
- Тап по строке → `/clients/:id` (карточка).
- Тап по FAB → `/clients/new` (форма создания).
- Никаких мутаций (чтение + навигация).

## Состояния

- **loading** (`clients.isPending`): «Загрузка…».
- **error** (`clients.isError`): «Не удалось загрузить клиентов. Попробуйте обновить страницу.» (role=alert).
- **empty** (`isSuccess && filtered.length===0`): при пустом запросе — «Пока нет клиентов.
  Добавьте первого.»; при непустом — «Никого не нашлось.».
- **Архивные клиенты** показываются в общем списке приглушёнными (отдельного раздела/фильтра «Архив» в UI нет).

## Навигация

строка→`/clients/:id` · FAB→`/clients/new` · назад→`/`.

## Бизнес-правила и edge-cases

- **Онлайн-статус** = поле `isOnline` клиента (формат работы), а НЕ presence: «Онлайн» = онлайн-формат, «Спортзал» = очно.
- Сортировка `session`: клиенты с ближайшим занятием идут вперёд (по date,time), без занятия — в группу «Без занятий».
- Заголовок группы: `date===today`→«Сегодня», `date===tomorrow`→«Завтра», иначе «<день> <месяц-род.>».
- `useSessions(today)` тянет занятия начиная с сегодня; фильтрация по `clientId` — на клиенте.
- Архив/активность переключается не здесь, а через `status` в правке (`PATCH`).

## Сводка эндпоинтов

- `GET /api/clients` → `{ clients: ClientResponse[] }` — список клиентов.
- `GET /api/sessions?from=<today>` → `{ sessions: SessionResponse[] }` — ближайшие занятия.
- `GET /api/files/:avatarFileId` — аватар (через защищённый роут файлов).
