# Сообщения — список диалогов (тренер) — MessagesPage.tsx

**Маршрут:** `/messages` · **Точки входа:** главная тренера (плитка «Сообщения»)
**Назначение:** список диалогов с клиентами по убыванию активности + поиск любого
активного клиента, чтобы написать ему (даже если диалога ещё нет).

## Макет (сверху вниз)

1. **ScreenHeader** — «Сообщения», назад → `/`.
2. **Поле поиска** (`bg-chip`, иконка Search): «Поиск клиента по имени или
   контакту». Крестик X очищает запрос (виден только при непустом).
3. **Тело** — два взаимоисключающих режима:
   - **Режим поиска** (`query` непуст): список найденных активных клиентов
     (Avatar 44px + имя 15px semibold + первый непустой контакт «тип: значение»
     12px muted + Chevron). Тап → чат. Если пусто — «Никого не найдено.».
   - **Режим диалогов** (`query` пуст): список диалогов. Каждая строка — Avatar
     44px (с **акцентной точкой** сверху-справа, если есть непрочитанные) + имя
     (bold при непрочитанных, иначе semibold) + mono-строка статуса (дата/время
     последнего сообщения «11 июня, 8:24», + «· N новых сообщений» при непрочитанных)
     - **акцентный бейдж-счётчик** непрочитанных справа, иначе Chevron.

## Данные

| Поле                 | Источник                                                         | Формат                                                                     |
| -------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------- |
| диалоги              | `useConversations()` → `GET /conversations` → `conversations[]`  | `ConversationResponse[]`                                                   |
| `conv.clientId`      | id клиента диалога                                               | —                                                                          |
| `conv.lastMessageAt` | время последнего сообщения (nullable)                            | «11 июня, 8:24» (год если не текущий) / «нет сообщений»                    |
| `conv.unreadCount`   | непрочитанные тренером входящие (от клиента)                     | бейдж + плюрализация «новое сообщение / новых сообщения / новых сообщений» |
| имя клиента          | `useClients()` → `GET /clients` → map по id → firstName/lastName | «Имя Фамилия» / «Клиент» если нет в map                                    |
| результаты поиска    | `clients.data`, фильтр `status==='active'` + `clientMatches`     | сортировка по имени (localeCompare)                                        |
| контакт в поиске     | первый `c.contacts` с непустым `value`                           | «type: value»                                                              |

`clientMatches`: совпадение по имени, по любому контакту (value/type) или по тегам
(`c.tags`), регистронезависимо; пустой запрос → всё совпадает.

Сортировка диалогов: по убыванию `lastMessageAt` (без сообщений — в конец).

## Действия

| Жест                                | Эффект                                  | API                         |
| ----------------------------------- | --------------------------------------- | --------------------------- |
| Ввод в поиск                        | Переключение в режим поиска по клиентам | — (клиентский фильтр)       |
| Тап X в поиске                      | Очистить запрос → режим диалогов        | —                           |
| Тап по диалогу / найденному клиенту | Перейти в чат                           | → `/clients/:clientId/chat` |

Мутаций нет — только чтение и навигация. Списки опрашиваются: диалоги каждые 8с,
клиенты — стандартно.

## Состояния

- **pending** (`conversations.isPending || clients.isPending`): «Загрузка…».
- **error** (`conversations.isError`): «Не удалось загрузить диалоги…» (role=alert).
- **empty диалоги** (`isSuccess && list.length===0`): иконка MessageSquare +
  «Пока нет диалогов с клиентами.».
- **empty поиск** (`searchResults.length===0`): «Никого не найдено.».
- **непрочитанные:** акцентная точка на аватаре + bold имя + бейдж-счётчик +
  mono-строка bold с хвостом «· N новых сообщений».

## Навигация

назад → `/` · диалог/клиент → `/clients/:clientId/chat`.

## Бизнес-правила и edge-cases

- В поиск попадают **только активные** клиенты (`status==='active'`), а диалоги
  показываются для любых (имя берётся из map клиентов; если клиента нет — «Клиент»).
- `unreadCount` = непрочитанные тренером **входящие** (от клиента).
- Год в штампе показывается только если он отличается от текущего.
- Диалоги без сообщений (`lastMessageAt===null`) → «нет сообщений», уходят в конец.
- Один acid-fill: акцент только на бейдже/точке непрочитанных.

## Сводка эндпоинтов

- `GET /conversations` — список диалогов (polling 8с) → `{conversations:[{id,clientId,lastMessageAt,unreadCount,createdAt}]}`.
- `GET /clients` — клиенты (имена для диалогов, источник поиска).
