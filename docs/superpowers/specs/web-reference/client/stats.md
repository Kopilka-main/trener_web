# Прогресс / Статистика клиента — StatsPage.tsx

**Маршрут:** `/progress` · **Точки входа:** плитка «Прогресс» с главной; глубокая
ссылка `?tab=measurements` (из задачи на замеры / уведомлений)
**Назначение:** прогресс клиента в трёх вкладках — упражнения (PR/тоннаж/тренд),
замеры тела (с аналитикой-графиками и формой ввода) и фото прогресса по ракурсам.

## Макет (сверху вниз)

1. **Заголовок** «Прогресс» (24px `font-display`).
2. **Сегмент-табы** (`bg-chip`, 3 кнопки): **Упражнения** / **Замеры** / **Фото**.
   Активный — `bg-card text-ink`, прочие — `text-ink-muted`. Начальная вкладка:
   `measurements` если `?tab=measurements`, иначе `exercises`.
3. **Контент вкладки** (`px-2 pb-8`):

### Вкладка «Упражнения»

- **Список упражнений** (`aggregateExerciseOverview(workouts)`): карточка-плитка
  на упражнение. Имя (15px semibold) + mono-строка метрик:
  - силовое: «PR <maxWeightKg> кг» · «тоннаж <formatTonnage>» · `formatRelativeDate(lastDate)`;
  - на время (`isTimeBased`): «PR <maxTimeSec>» · «время <totalTimeSec>».
  - Иконка тренда: `ArrowUp` accent-text если `lastIsRecord` (рекорд в последней
    сессии), иначе `ArrowDown` muted. Стрелка `ChevronRight`.
- **Детализация упражнения** (по тапу — `ExerciseDetail`, заменяет список):
  - «← Назад» + имя упражнения.
  - Тогл **«Только рекорды»** (по умолчанию ON) — фильтрует точки графиков по
    монотонно растущему максимуму.
  - **Два графика** `ChartCard` (SVG, интерактивный курсор «тяни по графику»):
    - силовое: «Тоннаж» (accent) и «Максимальный вес» (coral), ось Y — кг;
    - на время: «Максимальное время» (accent) и «Суммарное время» (coral), Y — сек.
    - Ось X — сессии по возрастанию даты; крупное число = значение под курсором
      (или последнее); под ним дельта к первой сессии «+N (+M%) с первой сессии»
      (зелёная при росте, danger при падении).
  - **Таблица «История»** (новые сверху): дата · N подх. · PR · тоннаж/«всего».

### Вкладка «Замеры»

- **Кнопка «Новый замер»** (пунктир, `Plus`) — только если `linked`.
- **Блок «Аналитика»** (если есть ≥1 замер):
  - **График динамики метрики** (`LineChart`) + чипы-переключатели метрик. Чип
    показывается, только если у метрики ≥2 непустых значения; активный чип —
    `bg-accent`. Метрики: Вес/Бицепс/Грудь/Под грудью/Талия/Живот/Ягодицы/Бёдра/
    Бедро/Голень/% жира. Ось X — даты замеров (возр.), Y — значение метрики.
  - **Мини-карта «Тоннаж по тренировкам»** (`LineChart`, accent) — тоннаж
    завершённых тренировок во времени; показывается только если ≥2 точки.
- **Карточки замеров** (новые сверху): дата (`formatRuDate`) + `Pencil` (правка);
  сетка 3 колонки только непустых полей (label + значение + суффикс); заметка.
- **Форма замера** (`MeasurementForm`, добавление/правка — заменяет список):
  Дата (`<input type=date>`) → Состав тела (Вес, % жира) → Обхваты (9 полей) →
  Заметка (textarea). Кнопка «Сохранить»; при правке слева `HoldToDelete`.

### Вкладка «Фото»

- **Карточка загрузки** (`bg-card`): `<input type=date>` (по умолчанию сегодня) +
  **`BodyPoseGuide`** (3 силуэта-кнопки выбора ракурса: Спереди/Сбоку/Сзади,
  активный — `border-accent bg-accent/10`; подпись «облегающая одежда, телефон на
  уровне пояса, ровный фон, вся фигура в кадре») + кнопка-`label` «Выбрать фото ·
  <ракурс>» (`<input type=file accept=image/*>`).
- **Группы по датам** (новые сверху): заголовок-дата + сетка 3 колонки квадратных
  превью. На превью: бейдж ракурса (mono, чёрная плашка) + `HoldToDelete` (справа-сверху).

## Данные

| Поле/блок          | Источник                                                      | Формат                    |
| ------------------ | ------------------------------------------------------------- | ------------------------- |
| `linked`           | `useClientMe()` → `link != null`                              | bool                      |
| упражнения (обзор) | `aggregateExerciseOverview(useClientWorkouts())`              | список `ExerciseOverview` |
| история упражнения | `aggregateExerciseHistory(workouts, exerciseId)`              | `ExerciseHistory`         |
| тоннаж тренировки  | `workoutRowStats(workout)` (только `status==='completed'`)    | кг                        |
| замеры             | `useClientMeasurements()` → `measurements[]` (новые→старые)   | `MeasurementResponse[]`   |
| фото               | `useClientProgressPhotos()` → `photos[]`                      | `PhotoResponse[]`         |
| src превью фото    | `fileUrl(p.file.id)` → `/api/client/files/:fileId`            | cookie-сессия             |
| ракурс фото        | `p.angle` ∈ `front`/`side`/`back` → ярлык Спереди/Сбоку/Сзади | —                         |

**Поля замера** (`MeasurementResponse`): `id`, `clientId`, `date` (YYYY-MM-DD),
`weightKg`, `bodyFatPct`, `bicepsCm`, `chestCm`, `underbustCm`, `waistCm`,
`bellyCm`, `glutesCm`, `hipsCm`, `thighCm`, `calfCm` — все `number|null`; `note`
(`string|null`, ≤2000), `createdAt`. В UI «Вес» кг, «% жира» %, обхваты см.

## Действия

| Жест/кнопка            | Эффект                         | API (метод путь, тело → ответ)                                                                                        | Инвалидация                                       |
| ---------------------- | ------------------------------ | --------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| Сменить таб            | локальный `tab`                | —                                                                                                                     | —                                                 |
| Тап по упражнению      | открыть детализацию (локально) | —                                                                                                                     | —                                                 |
| Тогл «Только рекорды»  | фильтр точек графика           | —                                                                                                                     | —                                                 |
| «Новый замер» / правка | открыть форму                  | —                                                                                                                     | —                                                 |
| Сохранить (создание)   | создать замер                  | `POST /api/client/measurements`, тело `CreateMeasurementRequest` → `{ measurement }`                                  | `client/measurements`, `client/measurement-tasks` |
| Сохранить (правка)     | обновить замер                 | `PATCH /api/client/measurements/:id`, тело частичный `UpdateMeasurementRequest` → `{ measurement }`                   | `client/measurements`, `client/measurement-tasks` |
| `HoldToDelete` замера  | удалить замер                  | `DELETE /api/client/measurements/:id` → `{ ok }`                                                                      | `client/measurements`                             |
| Выбрать фото           | загрузить фото                 | `POST /api/client/progress-photos` **multipart/form-data**: `photo`(File), `date`, `angle`, опц. `note` → `{ photo }` | `client/progress-photos`                          |
| `HoldToDelete` фото    | удалить фото                   | `DELETE /api/client/progress-photos/:id` → `{ ok }`                                                                   | `client/progress-photos`                          |

**Тело `CreateMeasurementRequest`** (PATCH — те же поля, все опциональны;
явный `null` = очистить поле): `date` (обяз., `^\d{4}-\d{2}-\d{2}$`), `weightKg`,
`bodyFatPct`, `bicepsCm`, `chestCm`, `underbustCm`, `waistCm`, `bellyCm`,
`glutesCm`, `hipsCm`, `thighCm`, `calfCm` (каждый `positive | null`), `note`
(`string ≤2000 | null`). Пустое поле в UI отправляется как `null`.

**Загрузка фото**: отдельный `fetch` с `credentials:'include'`, без ручного
`Content-Type` (boundary ставит браузер). `angle` — `front`/`side`/`back`.

## Состояния

- **Упражнения**: loading «Загрузка…»; error «Не удалось загрузить прогресс…»;
  empty — если `linked` «Тренировок с упражнениями пока нет», иначе CTA подключить
  тренера. Детализация без точек: «Это упражнение пока не встречалось…».
- **Замеры**: `!linked` → «Подключите тренера, чтобы вести замеры»; loading/error;
  empty (`linked`) «Замеров пока нет. Добавьте первый». Аналитика без ≥2 точек ни
  по одной метрике → «Недостаточно данных для графика…».
- **Фото**: `!linked` → «Подключитесь к тренеру, чтобы вести фото прогресса»;
  loading/error; empty «Фотографий пока нет». Ошибка загрузки → «Не удалось
  загрузить фото» (`role=alert`).
- 409 от любого из `measurements`/`progress-photos`/`trainer` (не привязан) → хук
  отдаёт пустой список/`null`, а не ошибку.

## Навигация

Экран самодостаточен (внутренние табы и detail-view). Внешних переходов нет —
возврат на главную через нижнюю навигацию / BackFab.

## Бизнес-правила и edge-cases

- **«Только рекорды»** (детализация упражнения): оставляет лишь точки, где
  значение превысило все предыдущие (монотонно растущий максимум).
- **Графики ChartCard** отбрасывают точки с `value<=0` и без даты.
- **Аналитика замеров**: метрика доступна для графика только при ≥2 непустых
  значениях; если выбранная метрика стала недоступной — авто-переключение на
  первую доступную; стартовая метрика — `weightKg`.
- **Тоннаж по тренировкам** учитывает только `status==='completed'`, сортирует по
  `completedAt ?? startedAt`, требует ≥2 точек.
- Список замеров приходит новые→старые; для графиков разворачивается по
  возрастанию даты.
- Форматирование: тоннаж ≥1000 кг → «N,N т»; время ≥3600 c → «ч», ≥60 → «мин».
- Удаление (замер/фото) — через **`HoldToDelete`** (удержание), без отдельного
  диалога подтверждения.

## Сводка эндпоинтов

- `GET /api/client/auth/me` — `link` (привязан?).
- `GET /api/client/workouts` — тренировки (обзор упражнений, тоннаж).
- `GET /api/client/measurements` — список замеров.
- `POST /api/client/measurements` — создать замер.
- `PATCH /api/client/measurements/:id` — обновить замер.
- `DELETE /api/client/measurements/:id` — удалить замер.
- `GET /api/client/progress-photos` — список фото.
- `POST /api/client/progress-photos` — загрузить фото (multipart).
- `DELETE /api/client/progress-photos/:id` — удалить фото.
- `GET /api/client/files/:fileId` — приватный файл фото (для `<img>`).
