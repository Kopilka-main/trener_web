# Главная клиента — HomePage.tsx

**Маршрут:** `/` · **Точки входа:** корневой после логина
**Назначение:** дашборд клиента — баланс тренировок, ближайшее занятие, 6 плиток-разделов.

## Макет (сверху вниз)

1. **Шапка тренера** (только если `linked && trainer`): аватар 36×36 (фото
   `/api/client/trainer/avatar?v=<avatarFileId>` или инициалы на `card-elevated`)
   - имя (14px semibold) + «тренер» (11px muted). Тап → `/trainer`.
2. **Шестерёнка** (абсолют, справа-сверху): иконка Settings 30px. Тап → `/profile`.
3. **Герой:**
   - Если **подключён**: display-число 64px (`font-display`) — `paidBalance`
     (оплаченные тренировки), цвет accent, при отрицательном — danger и без
     pad2. Рядом «количество тренировок» (22px bold). Тап → `/profile`.
     - Под ним (если есть `packageEndsAt`): иконка Flame + «до <дата>» (13px muted).
     - Строка ближайшего занятия (если есть `nextSession`): mono-кикер
       «СЛЕД. · <время> · <TITLE>» + акцентная плашка `diffShort` (через сколько)
       - стрелка `→`. Тап → `/calendar`.
   - Если **не подключён**: display-30px «Подключите тренера» (accent) + подпись
     «чтобы видеть занятия и прогресс» + `→`. Тап → `/connect`.
4. **Сетка плиток 2×3** (`grid-cols-2 grid-rows-3`), 6 плиток (см. ниже).

## Данные

| Поле               | Источник                                                                                                                       | Формат                          |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------ | ------------------------------- |
| `linked`           | `useClientMe()` → `link != null`                                                                                               | bool                            |
| тренер (шапка)     | `useClientTrainer()` → firstName/lastName/avatarFileId                                                                         | —                               |
| `paidBalance`      | Σ активных пакетов `lessonsPaid` − завершённые тренерские тренировки (`completed && !createdByClient && !excludedFromBalance`) | pad2/число                      |
| `packageEndsAt`    | самая поздняя `endsAt` среди активных пакетов                                                                                  | «31 дек 2026»                   |
| `nextSession`      | ближайшая будущая не-cancelled сессия (30 дней)                                                                                | время+title                     |
| плитка Тренировки  | `useClientWorkouts().length`                                                                                                   | pad2 «завершено»                |
| плитка Календарь   | сессии не-cancelled на 30 дней                                                                                                 | pad2 «на 30 дней»               |
| плитка Чат         | `useClientChatUnread()`                                                                                                        | pad2 «новых»                    |
| плитка Прогресс    | `recordsCount` = упражнения с `lastIsRecord` из обзора                                                                         | pad2 «рекордов»                 |
| плитка База знаний | `aggregateExerciseOverview(workouts).length`                                                                                   | pad2 «упражнений»               |
| плитка Уведомления | `attention` = прочие уведомления + открытые задачи                                                                             | pad2 «новых» / кикер «ВСЁ ТИХО» |

## Действия

- Тап по тренеру → `/trainer`.
- Тап по шестерёнке / герою-балансу → `/profile`.
- Тап по строке занятия → `/calendar`.
- Тап по «Подключите тренера» → `/connect`.
- Тап по плиткам → `/workouts`, `/calendar`, `/chat`, `/progress`, `/knowledge`,
  `/notifications`.
- Никаких мутаций (только чтение + навигация).

## Состояния

- **Не подключён** (`!linked`): герой = CTA подключения; шапки тренера нет;
  плитки показываются (счётчики по своим данным, многие 0).
- **Подключён**: герой = баланс + дата окончания + след. занятие.
- `paidBalance < 0`: число красным (danger), без ведущего нуля.
- Нет `nextSession`: строка занятия скрыта.
- **Primary-плитка** (один acid-fill): `unread>0` → «Чат»; иначе `attention>0` →
  «Уведомления»; иначе ни одна.

## Навигация

тренер→/trainer · профиль/баланс→/profile · занятие→/calendar · CTA→/connect ·
плитки→/workouts /calendar /chat /progress /knowledge /notifications.

## Бизнес-правила и edge-cases

- **Баланс = оплачено − проведено тренером.** Самостоятельные тренировки клиента
  (`createdByClient`) и `excludedFromBalance` в остаток НЕ входят.
- `diffShort`: `СЕЙЧАС` если прошло; `<d>Д` / `<d>Д <h>Ч` если ≥24ч; иначе
  `<h>Ч <m>М` / `<m>М` / `<h>Ч`.
- Метрики плиток умеют ротироваться (если несколько значений, смена раз в 10с) —
  здесь у каждой по одному значению.
- pad2: число < 10 показывается как «0N».

## Сводка эндпоинтов

- `GET /api/client/auth/me` — `link` (подключён?).
- `GET /api/client/trainer` — профиль тренера для шапки.
- `GET /api/client/sessions?from&to` — сессии на 30 дней.
- `GET /api/client/workouts` — тренировки (баланс, плитки).
- `GET /api/client/chat/unread` — непрочитанные.
- `GET /api/client/chat/messages` — открытые задачи.
- `GET /api/client/packages` — пакеты (баланс, дата окончания).
