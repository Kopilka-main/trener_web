# Календарь клиента — CalendarPage.tsx

**Маршрут:** `/calendar` · **Точки входа:** плитка «Календарь» с главной, строка ближайшего занятия в герое главной, нижняя навигация.
**Назначение:** показывать назначенные тренером занятия (день/неделя/месяц) и давать клиенту подтвердить или отклонить занятие.

## Макет (сверху вниз)

1. **Заголовок** «Календарь» — `font-display` 24px, `text-ink`, отступы `px-2 pt-5 pb-1`.
2. **Тело** — ветвится по состоянию (см. Состояния):
   - не подключён → текст-заглушка;
   - ошибка загрузки → текст-алерт;
   - иначе → компонент `SessionsCalendar` (стартовый вид — **неделя**).
3. **`SessionsCalendar`** (внутри):
   - **Шапка периода** (`px-2 pb-2`): кнопка `‹` (ChevronLeft 20px) · подпись периода по центру (15px semibold, `text-ink`) · кнопка `›` (ChevronRight 20px) · чип «Сегодня» (`bg-chip`, 12px, `text-ink-muted`).
   - **Область вида**: MonthView / WeekView / DayView (одна из трёх).
   - **Нижний переключатель вида** `ViewSwitcher` — `fixed bottom-4`, по центру (как FAB): пилюля `bg-card` с тремя кнопками «День / Неделя / Месяц»; активная — `bg-accent text-accent-on`.
4. **`SessionSheet`** (нижний лист, рендерится при выбранном занятии): затемнение `bg-black/60` + лист `rounded-t-3xl bg-bg`, прижат к низу с учётом `safe-area-inset-bottom`.

### Виды календаря

- **Месяц** (`MonthView`): сетка 6×7 от понедельника (`monthGrid`), шапка дней «Пн…Вс». Ячейка дня — `aspect-square rounded-xl bg-card`; сегодня — `ring-2 ring-accent`; дни вне месяца — `opacity-40`. В правом-верхнем углу при наличии занятий — акцентный четвертькруг. Под числом — счётчики по подтверждению (mono 10px): `pending` (нейтральный `text-ink-mutedxl`), `confirmed` (`text-accent-text`), `declined` (`text-danger`). Тап по дню → открыть этот день (вид «День»).
- **Неделя** (`WeekView`): бесконечная вертикальная лента недель (скользящее окно, как в iPhone). Липкая шапка «Пн…Вс». Каждая неделя — строка из 7 колонок-дней; в колонке стопка `WeekCard` (название до 2 строк + время начала, цвет по статусу). Сегодняшняя колонка подсвечена `bg-card/40`, сегодняшнее число — `bg-accent`. Тап по карточке → лист занятия; тап по числу или пустому месту колонки → открыть день.
- **День** (`DayView`): часовая сетка **00:00–24:00** (`CAL_START_HOUR=0`, `CAL_HOURS=24`), высота часа 56px. Левая колонка — метки часов (mono 10px). Занятия — абсолютные блоки по времени/длительности; пересекающиеся раскладываются по колонкам (`layoutColumns`). Линия «сейчас» (`NowLine`, `bg-coral`) — только если день сегодняшний. У клиента слоты-кнопки создания **выключены** (`onSlotClick` не передаётся) — рисуются просто часовые линии.

### Цвет блока занятия (`tileClasses`, приоритет сверху вниз)

| Условие                              | Цвет                                          | Смысл                                |
| ------------------------------------ | --------------------------------------------- | ------------------------------------ |
| `status === 'cancelled'`             | `bg-card-elevated`, перечёркнут, `opacity-50` | Отменено тренером                    |
| `clientConfirmation === 'confirmed'` | лайм `#caff3a`                                | Согласовано клиентом (план или факт) |
| `status === 'completed'`             | голубой `#46d4f0`                             | Проведено без согласования           |
| `clientConfirmation === 'declined'`  | красный `#ff5a5a`                             | Отклонено клиентом                   |
| иначе (`pending`)                    | оранжевый `#ffab2e`                           | Ждёт ответа                          |

### Лист занятия (`SessionSheet`)

- Заголовок: `title ?? 'Занятие'` (18px bold) + крестик «Закрыть».
- Строки: дата+время `«<день> <месяц-род>», <начало>–<конец>` (semibold) · человекочитаемая длительность (`humanDuration`, напр. «1 ч 45 мин») · если `isOnline` — иконка Wifi + «Онлайн-занятие», иначе `location` (если есть) · `note` (если есть).
- Плашка статуса (`bg-card`, 13px semibold): «Ожидает ответа» / «Вы подтвердили» / «Вы отклонили», либо «Отменено тренером» при `cancelled`.
- Кнопки **«Подтвердить»** (acid-fill `bg-accent`) и **«Отклонить»** (`bg-card`) — только если можно ответить (см. Бизнес-правила).
- При ошибке мутации — текст-алерт «Не удалось сохранить. Попробуйте снова.» (`text-ink-muted`, без красного).

Диапазон загрузки: вокруг опорной недели база `[anchor−3 нед .. anchor+5 нед]`, плюс скользящее окно недельной ленты (`onRangeChange` расширяет `from`/`to`; ISO-строки сравниваются лексикографически). Метка подписи занятия: `s.title ?? (trainerName || 'Занятие')`.

## Действия

| Жест                              | Эффект                                             | API                                                                                            |
| --------------------------------- | -------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| `‹` / `›`                         | сдвиг периода: день ±1, неделя ±14 дней, месяц ±1  | —                                                                                              |
| «Сегодня»                         | якорь = текущая дата                               | —                                                                                              |
| Переключатель «День/Неделя/Месяц» | смена вида (локально)                              | —                                                                                              |
| Тап по дню (месяц/неделя)         | открыть день (вид «День»)                          | —                                                                                              |
| Тап по занятию                    | открыть `SessionSheet`                             | —                                                                                              |
| Прокрутка недельной ленты         | догрузка занятий под видимое окно                  | `GET /api/client/sessions?from&to` (новый диапазон)                                            |
| **«Подтвердить»**                 | `clientConfirmation → confirmed`, лист закрывается | `POST /api/client/sessions/:id/confirmation`, тело `{ "status": "confirmed" }` → `{ session }` |
| **«Отклонить»**                   | `clientConfirmation → declined`, лист закрывается  | `POST /api/client/sessions/:id/confirmation`, тело `{ "status": "declined" }` → `{ session }`  |

**Инвалидации:** успех `useConfirmSession` инвалидирует все ключи `['client','sessions']` (все диапазоны списка) и закрывает лист (`onSuccess: onClose`).

## Состояния

- **loading**: список грузится — `SessionsCalendar` рендерится с пустым массивом (сетка без занятий); явного спиннера нет.
- **не подключён** (`!linked`): вместо календаря текст «Вы пока не подключены к тренеру. Подключите его, чтобы здесь появились назначенные занятия.» (`text-ink-muted`).
- **error** (`sessions.isError`): текст-алерт «Не удалось загрузить занятия. Попробуйте обновить страницу.» (`role="alert"`, `text-ink-muted`).
- **409 от API**: трактуется как пустой список (непривязанный клиент), НЕ ошибка — `useClientSessions` ловит `ApiError.status === 409` и возвращает `[]`.
- **empty** (подключён, занятий нет): календарь рисуется пустым (нет блоков/счётчиков/уголков).
- **отменённое занятие**: блок перечёркнут + `opacity-50`; в листе — «Отменено тренером», кнопок ответа нет.
- **мутация в процессе** (`confirm.isPending`): обе кнопки `disabled` + `opacity-50`.
- **ошибка мутации** (`confirm.isError`): алерт в листе, лист остаётся открытым.

## Навигация

- День/неделя/месяц и навигация периода — **внутри страницы**, без смены маршрута.
- Тап по дню → переключение на вид «День» (тот же `/calendar`).
- Тап по занятию → нижний лист (оверлей), не маршрут.
- Уход со страницы — через нижнюю навигацию приложения.

## Бизнес-правила и edge-cases

- **Когда можно ответить**: `canRespond = !cancelled && clientConfirmation === 'pending'`. Подтвердить/отклонить можно, даже если время начала уже прошло (факт-подтверждение); нельзя — если занятие отменено или уже есть ответ.
- **Тело запроса строго** `{ status: 'confirmed' | 'declined' }` (`clientSessionConfirmRequestSchema`); ответ — `{ session: SessionResponse }`.
- **Скоуп/безопасность**: сервер резолвит `trainerId`+`clientId` из сессии клиента; подтверждение чужого занятия → **404** (см. isolation-itest).
- **Polling 8 с** (вебсокетов нет) — занятия и подтверждения других обновляются опросом, как чат.
- **Автоскролл**:
  - day/week при монтаже/смене даты скроллят к **7:00** (`SCROLL_HOUR=7`, `ScrollableGrid`);
  - если в дне есть занятие — его блок дополнительно центрируется (`scrollIntoView({ block: 'center' })`);
  - недельная лента при `‹/›`/«Сегодня»/возврате из дня прыгает к опорной неделе (`block: 'start'`), при прокрутке удерживает позицию видимой недели.
- **Скользящее окно недель**: старт 20 недель (4 до опорной), докуп/очистка чанками по 6, потолок 44 недели в DOM; подгрузка при близости к краю < 800px.
- **Подпись периода**: день — «Понедельник, 5 июня»; неделя — диапазон 2 недель «2–15 июня» (или с двумя месяцами); месяц — «Июнь 2026».
- **Счётчики месяца** считаются по `clientConfirmation` (pending/confirmed/declined), а не по `status`.
- **Раскладка пересечений** (`layoutColumns`): занятия на одно время раздвигаются по колонкам, минимальная высота блока — 18px, длительность минимум 1 мин.

## Сводка эндпоинтов

- `GET /api/client/auth/me` — `link` (подключён ли клиент к тренеру).
- `GET /api/client/trainer` — имя тренера (фолбэк-подпись занятия).
- `GET /api/client/sessions?from&to` — список занятий за диапазон (polling 8 с; 409 → `[]`).
- `POST /api/client/sessions/:id/confirmation` — тело `{ status: 'confirmed' | 'declined' }` → `{ session }`; подтвердить/отклонить занятие (чужое → 404).
