# Тренер (карточка) — TrainerPage.tsx

**Маршрут:** `/trainer` · **Точки входа:** шапка/карточка тренера на главной и в
профиле
**Назначение:** публичный профиль привязанного тренера (имя, специализация, «о
себе», контакты) + отключение от тренера с подтверждением.

## Макет (сверху вниз)

1. **Заголовок** «Тренер» (24px `font-display`).
2. **Шапка тренера**: `TrainerAvatar` 64×64 (фото
   `/api/client/trainer/avatar?v=<avatarFileId>` или инициалы на `bg-chip`,
   фолбэк на инициалы при ошибке загрузки `onError`) + имя «Имя Фамилия» (20px
   bold) + `title` (специализация, muted, если есть).
3. **«О тренере»** (если `bio`) — заголовок-кикер + текст (`whitespace-pre-wrap`).
4. **«Контакты»** (если `contacts.length>0`) — список `bg-card` с разделителями:
   тип ↔ значение.
5. **`DisconnectSection`** — кнопка/подтверждение отключения (см. ниже).

## Данные

| Поле                   | Источник                                               | Формат              |
| ---------------------- | ------------------------------------------------------ | ------------------- |
| тренер                 | `useClientTrainer()` → `TrainerPublicResponse \| null` | —                   |
| `firstName`/`lastName` | `t.firstName`/`t.lastName`                             | имя/инициалы        |
| `title`                | `t.title`                                              | специализация       |
| `bio`                  | `t.bio`                                                | многострочный текст |
| `contacts`             | `t.contacts[]` (`{type,value}`)                        | список              |
| аватар                 | `t.avatarFileId` → `/api/client/trainer/avatar?v=<id>` | фото/инициалы       |

Профиль **публичный** — без email и приватных полей.

## Действия

| Жест/кнопка              | Эффект                                                     | API (метод путь, тело → ответ)                        | Инвалидация                   |
| ------------------------ | ---------------------------------------------------------- | ----------------------------------------------------- | ----------------------------- |
| «Отключиться от тренера» | раскрыть блок подтверждения                                | —                                                     | —                             |
| Ввод имени тренера       | разблокировать кнопку при точном совпадении (без регистра) | —                                                     | —                             |
| «Отмена»                 | свернуть блок, очистить ввод                               | —                                                     | —                             |
| «Отключиться»            | мягкая отвязка                                             | `POST /api/client/trainer/disconnect` → `{ ok:true }` | `client/me`, `client/trainer` |

Тело запроса отключения отсутствует (пустой POST).

## Состояния

- `trainer.isLoading` → «Загрузка…».
- Не подключён (`!t`, в т.ч. 409 → `null`) → «Тренер не подключён.».
- Подключён → шапка + опц. «О тренере»/«Контакты» + `DisconnectSection`.
- **Disconnect свёрнут**: одна кнопка «Отключиться от тренера».
- **Disconnect раскрыт**: пояснение (тренировки/замеры/история сохранятся) +
  поле ввода имени + «Отмена»/«Отключиться». Кнопка «Отключиться» `disabled`,
  пока введённое имя ≠ имени тренера (сравнение `.trim().toLowerCase()`).
- `disconnect.isError` → «Не удалось отключиться. Попробуйте снова.» (`role=alert`).
- `disconnect.isPending` → «Отключение…», кнопки `disabled`.

## Навигация

Возврат назад — `BackFab` (нижняя кнопка). После успешного отключения
`client/me` инвалидируется → роутинг уводит на экран «Подключение» (клиент стал
«не подключён»).

## Бизнес-правила и edge-cases

- **Отключение — мягкая отвязка**: рвётся только связь клиент↔тренер; тренировки,
  замеры, история, фото клиента **сохраняются** (явно сказано в пояснении).
- **Подтверждение вводом имени**: точное совпадение «Имя Фамилия» без учёта
  регистра — защита от случайного отключения.
- Кнопка «Отключиться» — единственное «опасное действие» на экране (`bg-danger`,
  белый текст), допустимо по правилу «красный только для кнопок реального действия».
- Аватар тренера: кэш-бастинг `?v=<avatarFileId>`; при ошибке загрузки — фолбэк
  на инициалы.

## Сводка эндпоинтов

- `GET /api/client/trainer` — публичный профиль тренера (409 если не привязан → null).
- `POST /api/client/trainer/disconnect` — отключение от тренера.
- `GET /api/client/trainer/avatar?v=<id>` — изображение аватара тренера.

## Расхождения мобайла (на момент составления)

- [P1] Подтверждение отключения **вводом имени тренера** (точное совпадение,
  без регистра) — проверить наличие в мобайле.
- [P2] Текст-гарантия «тренировки/замеры/история сохранятся» и фолбэк аватара
  на инициалы при ошибке загрузки.
