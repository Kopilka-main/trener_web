# Подключение тренера — ConnectPage.tsx

**Маршрут:** `/connect` · **Точки входа:** баннер `ConnectBanner` («Подключить тренера»), видимый в шапке приложения пока клиент не привязан (`link === null`); CTA «Подключите тренера» с главной (`/`).
**Назначение:** показать клиенту его код (= id аккаунта) и QR, чтобы тренер подключил клиента; экран сам сменится на приложение после привязки.

## Макет (сверху вниз)

Контейнер: `flex-1`, по центру по вертикали, `gap-6`, `px-2 py-8`, `text-center`.

1. **Заголовок** «Подключение» — `font-display`, 28px, `leading-tight`, `accent-text`.
2. **Подпись** (14px `ink-muted`): «Передай этот код тренеру — он подключит тебя, и появятся назначенные тренировки. Можно продолжить и заниматься самостоятельно.»
3. **QR-код**: блок `rounded-2xl bg-ink p-4`, по центру; `QRCodeSVG` (`qrcode.react`), `size=180`, `bgColor=#eeeee8`, `fgColor=#0b0c10`, значение = `code`.
4. **Блок кода** (`flex-col gap-1.5`):
   - **Кнопка-код** (`button`, `aria-label="Скопировать код"`): `rounded-xl border-line bg-chip`, `font-mono` 14px, текст `ink`, `active:bg-card-elevated`. Внутри — сам код (`break-all`) + иконка: `Copy` (16px, `ink-muted`) либо `Check` (16px, `accent-text`) после копирования.
   - **Подсказка** (12px `ink-muted`): «Нажми, чтобы скопировать» / «Скопировано».
5. **Кнопка «Продолжить»**: `rounded-xl bg-accent`, текст `accent-on`, semibold, `py-3`, `active:opacity-90`.

## Данные

| Поле     | Источник                                                           | Формат                  |
| -------- | ------------------------------------------------------------------ | ----------------------- |
| `code`   | проп от `App`: `me.data.account.id` (из `GET /api/client/auth/me`) | строка-id аккаунта      |
| `copied` | локальный `useState`                                               | bool (сброс через 1.5с) |

Экран сам не делает запросов — `code` приходит пропсом из уже загруженного `useClientMe`.

## Действия

- **Тап по кнопке-коду** (`copyCode`): `navigator.clipboard.writeText(code)` (ошибки буфера проглатываются); `setCopied(true)`; через `setTimeout` 1500мс → `setCopied(false)`. Иконка меняется `Copy` → `Check`, подпись «Скопировано».
- **Тап «Продолжить»**: `navigate('/')` — уход на главную (можно заниматься самостоятельно, не дожидаясь привязки).
- Никаких мутаций/POST. Привязку выполняет тренер на своей стороне.

## Состояния

- **Default**: код + QR + «Нажми, чтобы скопировать».
- **Copied** (1.5с): иконка `Check` (`accent-text`), подпись «Скопировано».
- **Привязка произошла** (тренер подключил клиента): `useClientMe` в `App` поллит `/me` каждые 4с пока `link === null`; как только `link` становится не-`null`, баннер `ConnectBanner` исчезает, приложение получает тренерские данные. Сам экран `/connect` остаётся доступным, но обычно клиент уже ушёл «Продолжить».
- Нет empty/error-веток — код всегда есть (иначе экран бы не отрендерился, ведь требуется залогиненный `me`).

## Навигация

- «Продолжить» → `/`.
- Доступ на экран: `/connect` из `ConnectBanner` или CTA на `/`.

## Бизнес-правила и edge-cases

- **Код = id аккаунта клиента** (`account.id`). Это и значение QR, и текст для ручной передачи — тренер вводит/сканирует его, чтобы создать связь.
- Привязка односторонняя со стороны тренера; клиентский экран лишь отдаёт код и пассивно ждёт (поллинг `/me` в `App`, а не на этом экране).
- Таймер копирования чистится при повторном тапе (`clearTimeout`) — нет накопления.
- Сбой буфера обмена (нет разрешения/HTTP) не показывает ошибку: `.catch(() => undefined)`, но `copied` всё равно ставится в `true` (UX-оптимистично).
- Экран доступен только залогиненному (роут внутри ветки `me.data` в `App`); неавторизованный сюда не попадёт.

## Сводка эндпоинтов

- Прямых вызовов нет.
- `GET /api/client/auth/me` — косвенно: источник `code` (поле `account.id`) и механизм авто-смены экрана после привязки (поллинг при `link === null`, интервал 4с, в `useClientMe`).
