# База знаний клиента — KnowledgePage.tsx

**Маршрут:** `/knowledge` · **Точки входа:** плитка «База знаний» с главной (`/`)
**Назначение:** read-only зеркало тренерской базы — поиск, две вкладки
(Тренировки / Упражнения), фильтры по группам и подгруппам мышц.

## Макет (сверху вниз)

1. **Заголовок** «База знаний» (28px `font-display`).
2. **Поиск** (`type=search`, card-фон, скруглённый 2xl): иконка `Search` слева,
   плейсхолдер «Поиск тренировок, упражнений». При непустом — кнопка `X`
   справа (очистить). Фильтрует по `name` активной вкладки (lowercase includes).
3. **Сегмент-табы** (контейнер `card-elevated`, два `SegmentTab`): «Тренировки» /
   «Упражнения». Активный — fill `accent` + `accent-on`; неактивный — `ink-muted`.
   Переключение вкладки сбрасывает выбранную группу и подгруппу.
4. **Чипы групп мышц** (`Chip`, mono 12px, `flex-wrap`): первый чип «Все»
   (`group===''`), далее группы из `orderGroups(present)`. Показываются только
   если набор непустой. Активный чип — fill `accent`.
5. **Чипы подгрупп** (вторая строка, только если выбрана группа и есть подгруппы):
   «Все» + подгруппы из `orderSubgroups(group, present)`.
6. **Список** (scroll-область):
   - Вкладка **Тренировки** — строки `card`: круг-бейдж с числом упражнений
     (mono bold), название (15px semibold, truncate), мета-строка mono
     «`<кратк. дата> · N упр.`», стрелка `ChevronRight`.
   - Вкладка **Упражнения** — строки `card` (min-h 84px): слева превью
     `ExerciseThumb` (бокс w-28, `object-contain`), справа название (line-clamp-2),
     подпись «`Группа · Подгруппа`», mono-строка «`PR <значение> кг|с · <дата>`».

## Данные

| Поле / элемент          | Источник                                                                  | Формат                         |
| ----------------------- | ------------------------------------------------------------------------- | ------------------------------ |
| `linked`                | `useClientMe()` → `link != null`                                          | bool (тексты пустых состояний) |
| каталог (обогащение)    | `useClientExercises()` → `Map<id, ExerciseResponse>`                      | имя/группа/подгруппа/медиа     |
| тренировки (вкладка)    | `useClientWorkouts()`, фильтр `!createdByClient`, сорт по `completedAt` ↓ | список                         |
| число упражнений строки | `w.exercises.length`                                                      | bold                           |
| дата строки             | `shortDate(w.completedAt)`                                                | «12 июн»                       |
| группы тренировок       | `category` упражнений тренировки из каталога → `orderGroups`              | чипы                           |
| подгруппы тренировок    | `subgroup` упражнений выбранной группы → `orderSubgroups`                 | чипы                           |
| упражнения (вкладка)    | `aggregateExerciseOverview(workouts)` + обогащение каталогом              | список                         |
| имя упражнения          | `catalog.name ?? overview.name`                                           | строка                         |
| категория / подгруппа   | `catalog.category` / `catalog.subgroup` (null — скрыто)                   | «Группа · Подгруппа»           |
| превью                  | `thumbUrl ?? imageUrl` (оба из каталога; null → плейсхолдер)              | картинка/иконка `Dumbbell`     |
| PR строки               | `overview.isTimeBased ? maxTimeSec+' с' : maxWeightKg+' кг'`              | mono, `b` ink                  |
| дата последней сессии   | `shortDate(overview.lastDate)`                                            | «· 12 июн»                     |

## Действия

- Ввод в поиск → локальная фильтрация (без запроса).
- Тап по чипу группы → `setGroup`, сброс подгруппы.
- Тап по чипу подгруппы → `setSubgroup`.
- Тап по строке тренировки → `/workouts/:id`.
- Тап по строке упражнения → `/knowledge/:exerciseId` (деталь упражнения).
- Мутаций нет (только чтение + навигация).

## Состояния

- **loading** (`workouts.isLoading || exercises.isLoading`): «Загрузка…».
- **error** (`workouts.isError || exercises.isError`): «Не удалось загрузить.
  Попробуйте обновить страницу.» (`role=alert`, нейтральный muted-текст).
- **empty — Тренировки:** если `trainerWorkouts` пуст и `linked` → «Пока нет
  тренировок от тренера.»; не `linked` → «Подключите тренера — здесь появятся
  проведённые им тренировки.»; если непуст, но фильтр пуст → «Ничего не нашлось.»
- **empty — Упражнения:** аналогично: «Пока нет упражнений из проведённых
  тренировок.» / «Подключите тренера — здесь появятся упражнения с ваших
  тренировок.» / «Ничего не нашлось.»
- **Без привязки тренера** (409 NOT_LINKED): хуки отдают пустые списки, не ошибку.
- Чипы групп/подгрупп скрыты, если набор пуст.

## Навигация

строка тренировки → `/workouts/:id` · строка упражнения → `/knowledge/:exerciseId`.

## Бизнес-правила и edge-cases

- **Вкладка «Тренировки» — только проведённые тренером:** `!w.createdByClient`
  (самостоятельные тренировки клиента исключены). Сортировка по `completedAt` ↓.
- **Группы/подгруппы тренировки** не хранятся в самой тренировке — берутся из
  каталога тренера по `exerciseId` каждого упражнения. Фильтр группы: тренировка
  проходит, если хоть одно её упражнение в этой группе (и подгруппе, либо без
  подгруппы при выбранной).
- **Вкладка «Упражнения»** строится из `aggregateExerciseOverview` (только
  завершённые тренировки), сортировка по дате последней сессии ↓.
- `orderGroups`: фикс-порядок `GROUP_ORDER` (Грудь, Спина, Ноги, Плечи, Руки,
  Корпус, Пресс/Кор, Кардио, Растяжка, Йога), остальные — по алфавиту (ru).
- `orderSubgroups(group, present)`: порядок по таксономии `SUBGROUPS_BY_GROUP`,
  неизвестные — в конец по алфавиту.
- **Превью**: предпочитается `thumbUrl`, фолбэк `imageUrl`; при ошибке загрузки
  `<img>` (`onError`) и при отсутствии URL — плейсхолдер `Dumbbell` на `bg-chip`.
  Картинка `object-contain` (вписывается, строку не растягивает).
- PR-строка: для «упражнений на время» (`isTimeBased`) показывается `maxTimeSec`
  в секундах, иначе `maxWeightKg` в кг; значения `null` → блок скрыт.
- Поиск и фильтры — чисто клиентские (по уже загруженным данным).

## Сводка эндпоинтов

- `GET /api/client/workouts` → `{ workouts: WorkoutResponse[] }` — тренировки
  клиента (источник вкладок и обзора упражнений). 409 NOT_LINKED → `[]`.
- `GET /api/client/exercises` → `{ exercises: ExerciseResponse[] }` — каталог
  тренера (имя/группа/подгруппа/медиа для обогащения). 409 NOT_LINKED → `[]`.
- `GET /api/client/auth/me` → `{ link }` — флаг подключения тренера (тексты
  пустых состояний).

### Поля `ExerciseResponse` (каталог)

`id, isGlobal, name, category, subgroup, description, defaultReps,
defaultWeightKg, defaultTimeSec, restSec, note, imageUrl, thumbUrl, videoUrl,
equipment, primaryMuscles, secondaryMuscles`. На этом экране используются:
`id, name, category, subgroup, imageUrl, thumbUrl`.

### Поля `WorkoutResponse` (использовано здесь)

`id, name, status, completedAt, createdByClient, exercises[]`; у упражнения —
`exerciseId, exerciseName, sets[]`.
