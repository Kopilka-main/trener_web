# Список тренировок клиента — WorkoutsListPage.tsx

**Маршрут:** `/workouts` · **Точки входа:** плитка «Тренировки» с главной (`/`), нижняя навигация.
**Назначение:** хаб тренировок клиента — текущая/новая тренировка, назначенные тренером, история завершённых, пикеры «Выбрать из базы» (шаблоны) и «Повторить из истории».

## Макет (сверху вниз)

1. **Заголовок** «Тренировки» (`font-display`, 28px, `ink`).
2. **Верхний блок — одно из трёх (по приоритету):**
   - `!linked` → текст-подсказка (`ink-muted`): «Вы пока не подключены к тренеру…».
   - есть `current` (тренировка со `status === 'active'`) → **ContinueCard**: имя (18px bold), «N упр. · идёт» (mono, muted), большая акцентная кнопка `Play` «Продолжить». Тап → `/workouts/:id/run`.
   - иначе → **NewWorkoutCard**: пунктирная карточка `border-dashed`, иконка `Plus` в круге, «Тренировка не запланирована», акцентная кнопка «Выбрать из базы» (→ пикер шаблонов) + текст-ссылка «или повторить из истории» (`RotateCcw`, disabled если истории нет).
3. **Загрузка / ошибка** (`q.isLoading` / `q.isError`): строки `ink-muted`.
4. **Секция «Назначено тренером»** (если есть): кикер uppercase (`ink-mutedxl`), карточки `bg-card` с именем + «N упр.» + `ChevronRight`. Тап → `/workouts/:id` (деталь/просмотр плана). Это черновики тренера (`!createdByClient && status === 'draft'`).
5. **Секция «Завершённые»** (если есть): кикер + группы по дате (`formatDateGroup`), внутри — список **HistoryRow**.

### HistoryRow (карточка завершённой)

- Слева: имя тренировки + чип «своя»/«от тренера» (по `createdByClient`); под ним mono-мета: время (`formatTime`) · `exercisesText` («N упр.» или «N из M упр.») · «M мин» (`durationSec/60`) · «RPE N».
- Кнопка `RotateCcw` (повторить) — disabled, если ни один подход не `done` (`canRepeat`).
- Кнопка `ChevronDown/Up` (развернуть).
- **Развёрнуто:** строки упражнений (метка `exerciseLabels` нумерует дубликаты «Имя 1/2» + `exerciseSummary` = «3× 12 × 50 кг»), `trainerNote` курсивом, кнопка «Сохранить как шаблон» (→ `POST /client/templates`, состояния «Сохраняем…» / «В шаблонах ✓»).

## Состояния

- **loading:** «Загрузка…».
- **error:** «Не удалось загрузить. Потяните обновить.».
- **empty / не подключён:** `!linked` → подсказка-CTA, истории/назначений нет (workouts вернут `[]` при 409 NOT_LINKED, без ошибки).
- **active есть:** показывается «Продолжить», блок выбора новой скрыт (как у тренера — пока идёт, не выбираем новую).
- **пикер шаблонов пуст:** текст «Шаблонов пока нет…».
- **HistoryRow без `done`-подходов:** повтор disabled (`canRepeat=false`).

## Навигация

ContinueCard / repeat / выбор шаблона → `/workouts/:id/run` · карточка «Назначено тренером» → `/workouts/:id`.

## Бизнес-правила и edge-cases

- **Черновик — временное превью.** `createDraftAndOpen` создаёт `draft` и сразу открывает RunWorkout (`DraftView`). Если уйти, не нажав «Начать», черновик удаляется (см. run-workout.md). В списке черновики НЕ показываются (кроме `assigned` от тренера).
- **`repeatBody`** (повтор «точь-в-точь»): берёт только `done`-подходы, мапит **факт → план** (`actualReps ?? plannedReps` и т.д.), пропущенные подходы исключаются; упражнения без подходов отбрасываются.
- **`templateBody`** (сохранить как шаблон): берёт **план** (или факт как запасной), сохраняет всю структуру; упражнения без подходов отбрасываются.
- **`exercisesText`**: «N из M упр.», если выполнено не всё; иначе «M упр.».
- **`exerciseLabels`**: повторяющиеся имена упражнений нумеруются по `position`.
- 409 NOT_LINKED во всех трёх запросах (workouts/templates/exercises) → пустой список, не ошибка: приложением можно пользоваться самостоятельно.

## Сводка эндпоинтов

- `GET /api/client/auth/me` — `link` (подключён?).
- `GET /api/client/workouts` — список тренировок.
- `GET /api/client/templates` — свои шаблоны (для пикера).
- `POST /api/client/workouts` — создать черновик (выбор шаблона / повтор).
- `POST /api/client/templates` — сохранить тренировку как шаблон.
- `DELETE /api/client/templates/:id` — удалить шаблон.
