# Итоги тренировки — WorkoutDetailPage.tsx

**Маршрут:** `/workouts/:wid` · **Точки входа:** карточка «Назначено тренером» и редирект из RunWorkoutPage после завершения (`completed`/`skipped`).
**Назначение:** read-only просмотр одной тренировки — мета (дата, длительность, RPE), заметка тренера, упражнения с фактом/планом по подходам и пометкой рекордов. Без мутаций.

## Макет (сверху вниз)

1. Ссылка-назад «‹ Тренировки» (`ChevronLeft`, → `/workouts`).
2. **Header:**
   - Имя тренировки (`font-display`, 24px, `ink`).
   - Мета-строка (12px `ink-muted`): дата (`formatDateGroup(completedAt)`) `· M мин` (`durationSec/60`) `· RPE N` — каждый сегмент опционален.
   - `trainerNote` (если есть): плашка `bg-card`, 13px `ink-muted`.
3. **Список упражнений** (`w.exercises`): для каждого — секция `bg-card`:
   - Заголовок `exerciseName` (15px semibold).
   - Список подходов: слева **факт** (`factText`) + плашка «рекорд» (acid-fill), если подход в `recordKeys`; справа **план** (`planText`, 12px `ink-mutedxl`).

## Данные

| Поле                  | Источник                                                  | Формат                            |
| --------------------- | --------------------------------------------------------- | --------------------------------- |
| `w`                   | `useClientWorkout(wid)` → `WorkoutResponse`               | —                                 |
| список (для рекордов) | `useClientWorkouts()` → `WorkoutResponse[]`               | —                                 |
| `recordKeys`          | `computeRecordKeys(list)` → `Set<string>` ключей `setKey` | пометка «рекорд»                  |
| дата                  | `completedAt` → `formatDateGroup`                         | «20 июня»                         |
| длительность          | `durationSec` → `Math.round(/60)`                         | «48 мин»                          |
| `rpe`                 | `rpe`                                                     | «RPE 8»                           |
| факт подхода          | `factText(s)`                                             | «12 × 50 кг» / «45 сек» / «—»     |
| план подхода          | `planText(s)`                                             | «план 12 × 50 кг» / «план 45 сек» |

## Действия

- Тап «‹ Тренировки» → `/workouts`.
- Никаких мутаций (полностью read-only).

## Состояния

- **loading:** «Загрузка…».
- **error / не найдена:** «Тренировка не найдена.» + ссылка «К списку».
- **успех:** `WorkoutBody`.
- Нет `completedAt`/`durationSec`/`rpe` → соответствующий сегмент меты опущен.
- Нет `trainerNote` → плашка скрыта.

## Навигация

«‹ Тренировки» / «К списку» → `/workouts`.

## Бизнес-правила и edge-cases

- **`factText`:** приоритет — `actualTimeSec` («N сек»); иначе `actualReps × actualWeightKg кг` (любой `null` → «—» вместо числа / без веса); если факта нет совсем → «—».
- **`planText`:** аналогично, с префиксом «план »; время приоритетнее повторов/веса.
- **Рекорды (`computeRecordKeys`):** по всем завершённым тренировкам для каждого `exerciseId` ищется лучший подход — максимум по `actualWeightKg`, при равенстве по `actualReps`, затем по `actualTimeSec`. Подходы без фактических значений игнорируются. Ключ — `setKey(workoutId, position, setIndex)` = `"<id>:<position>:<setIndex>"`. Поэтому «рекорд» — глобальная пометка по истории, а не в рамках этой тренировки.
- Экран показывает любую тренировку по `:wid` (в т.ч. назначенный тренером черновик — тогда виден план, факт «—»), но из RunWorkoutPage сюда попадают только `completed`/`skipped`.

## Сводка эндпоинтов

- `GET /api/client/workouts/:wid` — тренировка (детали, подходы, заметка).
- `GET /api/client/workouts` — все тренировки (для вычисления рекордов).

## Расхождения мобайла (на момент составления)

- Свериться: пометка «рекорд» считается по ВСЕЙ истории (`computeRecordKeys`), а не внутри тренировки.
- Свериться: формат факт/план (время приоритетнее; «—» при отсутствии) и порядок сегментов меты.
