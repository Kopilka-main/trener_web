# Деталь упражнения клиента — ExerciseDetailPage.tsx

**Маршрут:** `/knowledge/:exerciseId` · **Точки входа:** строка упражнения на
вкладке «Упражнения» базы знаний (`/knowledge`)
**Назначение:** read-only карточка упражнения — описание и параметры из каталога
тренера + личный результат клиента (PR) из проведённых тренировок.

## Макет (сверху вниз)

1. **Заголовок** (`header`): название упражнения (24px `font-display`); под ним —
   подпись «`Категория · Подгруппа`» (13px muted), если хотя бы одно задано.
2. **Описание** (`section`): mono-кикер «ОПИСАНИЕ» (11px uppercase muted) +
   текст `description` (14px ink) или «Описание не задано», если null.
3. **Параметры** (`section`, только если есть хоть один ненулевой параметр):
   кикер «ПАРАМЕТРЫ» + список строк в `card` (label muted слева, value mono
   tabular справа): Повторы / Вес (кг) / Время (сек) / Отдых (сек).
4. **Ваш результат** (`section`, только если есть `overview`): кикер «ВАШ
   РЕЗУЛЬТАТ» + блок `card`: «PR вес `<maxWeightKg>` кг» **или** «PR время
   `<maxTimeSec>` сек» (по `isTimeBased`) + «Последняя тренировка `<дата>`».
5. **Назад** — глобальный `BackFab` (не часть этой страницы).

## Данные

| Поле              | Источник                                                        | Формат                       |
| ----------------- | --------------------------------------------------------------- | ---------------------------- |
| `entry` (каталог) | `useClientExercises()` → `find(e.id === exerciseId)`            | `ExerciseResponse \| null`   |
| `overview` (PR)   | `aggregateExerciseOverview(workouts).find(o.exerciseId === id)` | `ExerciseOverview \| null`   |
| name              | `entry.name ?? overview.name ?? ''`                             | «Упражнение», если пусто     |
| category/subgroup | `entry.category` / `entry.subgroup` (null — подпись скрыта)     | «Кат · Подгр»                |
| description       | `entry.description`                                             | текст / «Описание не задано» |
| Повторы           | `entry.defaultReps` (если `>0`)                                 | число                        |
| Вес               | `entry.defaultWeightKg` (если `>0`)                             | «<n> кг»                     |
| Время             | `entry.defaultTimeSec` (если `>0`)                              | «<n> сек»                    |
| Отдых             | `entry.restSec` (если `>0`)                                     | «<n> сек»                    |
| PR вес            | `overview.maxWeightKg` (если не null и `!isTimeBased`)          | «<n> кг»                     |
| PR время          | `overview.maxTimeSec` (если не null и `isTimeBased`)            | «<n> сек»                    |
| Последняя трен.   | `shortDate(overview.lastDate)`                                  | «12 июн»                     |

## Действия

- Только чтение. Никаких мутаций.
- Навигация назад — глобальный `BackFab` (история роутера).

## Состояния

- **loading** (`exercises.isLoading || workouts.isLoading`): «Загрузка…».
- **Нет данных каталога** (`entry === null`): имя берётся из `overview.name`,
  иначе «Упражнение»; секции «Параметры» нет (params пуст), описание — «Описание
  не задано».
- **Нет `overview`** (упражнение не делалось в завершённых тренировках): секция
  «Ваш результат» скрыта.
- **Параметры:** секция целиком скрыта, если ни один из 4 параметров не задан
  (>0).
- **Без привязки тренера** (409 NOT_LINKED): оба хука отдают пустые списки;
  при неизвестном `exerciseId` — заголовок «Упражнение», только пустое описание.

## Навигация

Входящий маршрут `/knowledge/:exerciseId`; выход — `BackFab` → предыдущий экран
(обычно `/knowledge`).

## Бизнес-правила и edge-cases

- **Два независимых источника:** справочные данные (описание, дефолтные
  параметры) — из каталога тренера (`entry`); личный результат (PR, дата) — из
  агрегата по завершённым тренировкам клиента (`overview`). Любой может
  отсутствовать.
- **`isTimeBased`** определяет, что считать рекордом: упражнение «на время»
  (если время-подходов больше, чем вес-подходов) → показывается PR времени,
  иначе PR веса.
- `paramRows` включает только заданные **положительные** значения (`> 0`).
- `shortDate`: «<день> <ru-месяц>» («12 июн»); пустая строка при невалидной дате.
- **Графика истории НЕТ.** Хотя `aggregateExerciseHistory` (точки по сессиям)
  существует в `lib/workout-stats.ts`, эта страница его не использует — показан
  только агрегат (PR + дата последней сессии), без диаграммы/прогресса по сессиям.
- **Медиа НЕ показывается.** `imageUrl` / `thumbUrl` / `videoUrl` и справочные
  поля каталога (`equipment`, `primaryMuscles`, `secondaryMuscles`, `note`,
  `isGlobal`) на этой странице не отображаются — только название, категория,
  подгруппа, описание, 4 параметра и блок «Ваш результат».

## Сводка эндпоинтов

- `GET /api/client/exercises` → `{ exercises: ExerciseResponse[] }` — каталог
  тренера; `entry` ищется по `exerciseId`. 409 NOT_LINKED → `[]`.
- `GET /api/client/workouts` → `{ workouts: WorkoutResponse[] }` — завершённые
  тренировки для агрегата PR. 409 NOT_LINKED → `[]`.

### `ExerciseResponse` — поля, используемые здесь

`name, category, subgroup, description, defaultReps, defaultWeightKg,
defaultTimeSec, restSec`. (Полная схема также содержит `id, isGlobal, note,
imageUrl, thumbUrl, videoUrl, equipment, primaryMuscles, secondaryMuscles` —
здесь не используются.)

### `ExerciseOverview` — поля, используемые здесь

`exerciseId, name, isTimeBased, maxWeightKg, maxTimeSec, lastDate`. (Также
доступны `tonnageKg, totalTimeSec, lastIsRecord` — здесь не показаны.)
