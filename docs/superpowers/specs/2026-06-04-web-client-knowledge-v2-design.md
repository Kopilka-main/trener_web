# База знаний клиента v2 — дизайн

**Дата:** 2026-06-04
**Раздел:** `apps/web-client` → «База знаний» (`/knowledge`) — обогащение каталогом, фильтры, деталь.
**Связано:** [[project_web_client_app]]. Предшествует фиче «Свои тренировки» (выбор упражнений отсюда).

## Цель

В «Базе знаний» клиент видит упражнения, которые тренер давал ему на тренировках, **с описанием,
группой мышц и подгруппой**; может **отфильтровать по группе/подгруппе** и **провалиться в упражнение**
(деталь read-only: описание + параметры, как у тренера) — и посмотреть свой результат по нему.

## Решения

1. **Охват — «только с моих тренировок»**: список = упражнения из завершённых тренировок клиента,
   обогащённые данными каталога тренера (описание/категория/подгруппа/дефолты). Если упражнения нет
   в каталоге — показываем по имени из тренировки (без описания).
2. **Каталог** тянем новым read-only фасадом `GET /api/client/exercises` (через `resolveScope` →
   `exercises.service.list(trainerId)`: глобальные + личные тренера). Новых таблиц/миграций нет.
3. Фильтр по **группе мышц** (категория) и **подгруппе**; деталь — отдельный экран read-only.

## Архитектура

### Бэкенд

**Фасад `client-app-exercises`** (6-й, паттерн `makeClientScope`), переиспользует `exercises`-сервис:

- `GET /api/client/exercises` → `{ exercises }` (через `scope(req)` → `svc.list(trainerId)`).
- 401 без `client_sid`; 409 NOT_LINKED для непривязанного. Регистрация в `app.ts`.
- Контракт уже есть: `exerciseListResponseSchema`/`exerciseResponseSchema` (`@trener/shared`) —
  поля `id, isGlobal, name, category, subgroup, description, defaultReps/WeightKg/TimeSec, restSec, note`.
- Isolation itest (trener_test): привязанный получает каталог тренера; без сессии 401; непривязанный 409.

### Клиентский фронт

- **Порт** `apps/web/src/lib/muscleGroups.ts` → `apps/web-client/src/lib/muscleGroups.ts`
  (`SUBGROUPS_BY_GROUP`, `subgroupsFor`, `orderSubgroups`) — для фильтра подгрупп.
- `api/exercises.ts`: `useClientExercises()` (`GET /api/client/exercises`, 409→[]).
- **`KnowledgePage` (переработка):**
  - `useClientWorkouts()` → `aggregateExerciseOverview` (уже есть): список упражнений с тренировок
    (id, name, PR, lastDate, isTimeBased) — это «охват».
  - `useClientExercises()` → `Map<id, ExerciseResponse>` (каталог: категория/подгруппа/описание/дефолты).
  - Объединение: для каждого упражнения из тренировок — карточка с названием, категорией·подгруппой
    (из каталога, если есть).
  - **Фильтры (внизу, one-handed):** чипы групп мышц (категории, реально присутствующие у упражнений
    клиента) + при выбранной группе — чипы подгрупп (`orderSubgroups(group, присутствующие)`). «Все» сбрасывает.
  - Тап по карточке → `/knowledge/:exerciseId`.
- **`ExerciseDetailPage` (`/knowledge/:exerciseId`, read-only):** название, «категория · подгруппа»,
  **описание**, параметры по умолчанию (повторы/вес/время/отдых — те, что заданы), и блок «Ваш результат»
  (PR/последняя дата + ссылка-намёк на Прогресс). Навигация назад — `BackFab`. Если упражнения нет в
  каталоге — показываем имя + «Описание не задано».
- Маршрут `/knowledge/:exerciseId` в `App.tsx`. Плитка «База знаний» на главной — без изменений
  (счётчик упражнений уже есть).

## Поток данных

1. `/knowledge`: параллельно `useClientWorkouts` + `useClientExercises`. Список = упражнения с тренировок,
   обогащённые каталогом; фильтры по группе/подгруппе.
2. Тап → `/knowledge/:id`: данные из каталога (`useClientExercises` по id) + результат из overview.

## Обработка ошибок / пустые состояния

- Непривязанный (409 → []): список пуст, подсказка «Подключите тренера…».
- Каталог не загрузился: показываем упражнения по имени из тренировок без описания (graceful).
- Упражнение не в каталоге: имя + «Описание не задано».
- Без красного текста.

## Тестирование

- Isolation itest фасада `client-app-exercises` (401/409/каталог).
- Unit: фильтрация/объединение (хелпер сборки списка с фильтром — чистая функция, если выделим).
- `KnowledgePage`/`ExerciseDetailPage`: рендер из мок-хуков (список, фильтр, деталь, пустые состояния).
- `npm run check` + сборка web-client зелёные.

## Вне scope

- Редактирование упражнений клиентом (read-only).
- «Свои тренировки» (отдельный следующий цикл) — но деталь/каталог здесь готовят почву для выбора упражнений.
- Видео/картинки упражнений (в каталоге их нет).
