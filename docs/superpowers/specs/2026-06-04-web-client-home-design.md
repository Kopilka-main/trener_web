# Главная клиентского приложения — дизайн

**Дата:** 2026-06-04
**Раздел:** `apps/web-client` → «Главная» (хаб-страница по образцу тренерской `apps/web/src/pages/HomePage.tsx`).
**Связано:** [[project_web_client_app]].

## Цель

Дать клиенту хаб-главную в визуальной модели тренерской `HomePage`: верхняя строка
(дата + шестерёнка), крупный «герой»-блок и сетка плиток 2×3 с метриками. Главная заменяет
текущий лендинг (список тренировок) на `/`. Новый бэкенд не нужен — все данные через
существующие клиентские хуки.

## Решения (зафиксированы в брейншторме)

1. **Структура — «Новая Главная + нижнее меню»**: `/` = новый хаб; список тренировок → `/workouts`.
   Нижнее меню сохраняется (5 вкладок): 1-я — «Главная» (Home), далее Календарь/Чат/Прогресс/Профиль.
   «Тренировки» — плиткой на хабе.
2. **Визуальное зеркало** тренерской `HomePage`: те же стили (`tile-shadow`/`tile-shadow-primary`,
   mono-метки, display-числа), герой-число + строка «следующей», сетка 2×3, одна primary-плитка.
3. **Данные клиентские**, новый бэкенд не нужен.

## Подход

Порт визуальных примитивов (`Tile`, `MetricLabel`, date-хелперы) из тренерской `HomePage.tsx`
в `apps/web-client` (приложения не импортируют друг друга), адаптация под клиентские плитки и
данные. Список тренировок остаётся прежним компонентом `WorkoutsListPage`, переезжает на `/workouts`.

## Архитектура

### Навигация (`apps/web-client/src/App.tsx`, `BottomNav.tsx`)

- Новый экран `HomePage` на `/`.
- `WorkoutsListPage` → маршрут `/workouts` (компонент без изменений логики).
- `BottomNav.ITEMS`: 1-й элемент `{ to: '/', label: 'Главная', Icon: Home, end: true }`; остальные
  (`/calendar`, `/chat`, `/progress`, `/profile`) без изменений. Бейдж непрочитанных остаётся на «Чат».
- Деталь тренировки `/workouts/:wid` — без изменений.

### Экран `HomePage` (`apps/web-client/src/pages/HomePage.tsx`)

**Верхняя строка**

- Слева — дата `СЕГОДНЯ · ПН 3 ИЮНЯ` (mono, uppercase), как у тренера.
- Справа — шестерёнка (`Settings`) → `/profile`.

**Герой-блок**

- Крупное число = количество **сессий сегодня** из `useClientSessions(from,to)` (online включительно —
  клиент посещает все свои занятия); подпись «сессия/сессий». Тап → `/calendar`.
- Строка «СЛЕД. · `HH:MM` · `НАЗВАНИЕ` `[diffShort]` →» — ближайшая будущая сессия
  (date+startTime ≥ now, не cancelled), бейдж обратного отсчёта; тап → `/calendar`.
- **Непривязанный клиент**: вместо чисел — CTA «Подключите тренера» → `/connect`. Глобальный
  `ConnectBanner` остаётся над контентом (как сейчас).

**Сетка плиток 2×3** (порт `Tile`): иконка, стрелка `ArrowUpRight`, крупная метрика + mono-подпись,
заголовок, подпись.

1. **Тренировки** — `pad2(completedCount)` «завершено» → `/workouts` (`useClientWorkouts`).
2. **Календарь** — `pad2(plannedNext30d)` «на 30 дней» → `/calendar` (`useClientSessions`,
   статус не cancelled, на 30 дней вперёд).
3. **Чат** — `pad2(unread)` «новых» → `/chat` (`useClientChatUnread`).
4. **Прогресс** — `pad2(measurementsCount)` «замеров» → `/progress` (`useClientMeasurements`).
5. **Тренер** — имя тренера (или «Подключить», если не привязан) → `/profile`
   (привязан) / `/connect` (не привязан) (`useClientTrainer`/`useClientMe`).
6. **Профиль** — имя клиента → `/profile` (`useClientMe`).

**Primary-плитка (один acid-fill, в порядке приоритета):**

1. `unread > 0` → «Чат» primary;
2. иначе есть будущая сессия с `clientConfirmation === 'pending'` → «Календарь» primary;
3. иначе не привязан → «Тренер» primary;
4. иначе — без primary.

### Переиспользование

- Порт из `apps/web/src/pages/HomePage.tsx`: компоненты `Tile`, `MetricLabel`, типы `Metric`/`TileProps`
  и date-хелперы (`isoDate`, `pad2`, `timeToMinutes`, `toLocalDate`, `diffShort`, `DAY_SHORT`,
  `MONTH_FULL`). Адаптировать список плиток и источники данных под клиента.
- Иконки `lucide-react`: `Home`, `Dumbbell`, `Calendar`/`CalendarDays`, `MessageSquare`,
  `TrendingUp`, `User`, `Settings`, `ArrowRight`, `ArrowUpRight`.

## Поток данных

1. `HomePage` монтируется → параллельно `useClientSessions(today, +30д)`, `useClientWorkouts`,
   `useClientChatUnread`, `useClientMeasurements`, `useClientMe`, `useClientTrainer`.
2. Герой: считает сессии сегодня и ближайшую будущую из списка сессий.
3. Плитки: метрики из соответствующих хуков; primary — по правилам выше.
4. Тапы — навигация по маршрутам.

## Обработка ошибок / пустые состояния

- Непривязанный (409 → хуки отдают `[]`/`0`): метрики `00`, герой — CTA подключения, плитка
  «Тренер» = «Подключить» (primary). Не ошибка.
- Загрузка: нейтральные значения/плейсхолдеры; без красного текста (правило проекта — красный
  только для иконок severity и кнопок реального действия).

## Тестирование

- Бэкенда нет → только фронт.
- Unit-тест `HomePage` (vitest + testing-library, как тренерская `HomePage.test.tsx`):
  - рендер героя при наличии сессий сегодня и строки «следующей»;
  - метрики плиток из мок-хуков;
  - выбор primary-плитки (чат при unread; календарь при pending; тренер при отсутствии привязки);
  - состояние «не привязан» (CTA вместо чисел).
- `npm run check` зелёный; типы/линт чистые.

## Вне scope

- Изменения тренерской `HomePage`.
- Новые эндпоинты / серверная агрегация.
- Уведомления/алерты (модуля нет — как и у тренера сейчас).
- Изменение логики `WorkoutsListPage` (только смена маршрута).
